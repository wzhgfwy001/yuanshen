# Cattle Policy（牛群策略）

> 基于 Anthropic Managed Agents 的 Pet vs Cattle 设计哲学
> 生成时间：2026-05-12

## 核心理念

- **Pet（宠物）**：精心维护、不能挂、挂了要救
- **Cattle（牛群）**：坏了就换、不需要救、自动替换

**我们的子Agent是"牛群"，不是"宠物"。**

## 自动化处理规则

### 失败类型 → 自动处理

| 失败类型 | 处理方式 | 重试次数 |
|---------|---------|---------|
| 子Agent超时 | 终止 + 替换新实例 | 最多3次 |
| 子Agent崩溃 | 自动创建新实例继续 | 最多3次 |
| 模型API失败 | 等待 + 重试 + 换模型 | 最多3次 |
| 工具调用失败 | 记录 + 跳过 + 继续其他 | 无限制 |

### 禁止行为
- ❌ 子Agent失败后不人工介入修复
- ❌ 不尝试"救活"崩溃的实例
- ✅ 直接创建新实例替换

## 恢复流程

```
子Agent失败
    ↓
检查失败类型
    ↓
重试1-3次
    ↓
仍失败 → 创建新实例 → 继续任务
    ↓
记录到 brain/tasks/recovery.md
```

## 状态报告

每次自动替换时，记录：
- 失败的Agent ID
- 失败原因
- 替换后的新Agent ID
- 任务是否继续成功

- 系统可靠性：99%+
