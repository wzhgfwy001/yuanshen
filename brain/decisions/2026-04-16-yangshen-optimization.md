## 决策：任务轮廓(TaskProfile)数据结构

### 背景

**问题：** 任务分类器→分解器之间存在"信息断层"
- 分类器只输出"类型标签"
- 分解器"盲分解"，缺少任务上下文

### 方案

创建TaskProfile数据结构，包含：
- 任务类型（simple/simple-response/innovative/hybrid）
- 复杂度评级（1-5级）
- 关键约束（质量要求、格式要求、特殊需求）
- 所需能力（需要哪些子Agent能力）
- 执行策略建议（串行/并行/混合）
- 任务分解结果（decomposition）

### 输出文件

| 文件 | 路径 | 状态 |
|------|------|------|
| Schema定义 | `core/task-classifier/task-profile.schema.json` | ✅ 完成 |
| 使用示例 | `core/task-classifier/task-profile.example.json` | ✅ 完成 |

### 对接方式

```
分类器输出 ──→ TaskProfile ──→ 分解器使用
    │                              │
    │  taskType                    │ 根据decomposition
    │  complexityLevel             │ 和executionStrategy
    │  constraints                 │ 执行分解
    │  requiredCapabilities        │
    └──────────────────────────────┘
```
