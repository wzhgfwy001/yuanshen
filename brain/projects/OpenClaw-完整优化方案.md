## 三、详细方案

### 3.1 方向一：自动触发（最核心）


> **问题根源：** 系统有并行能力，但需要用户提醒才会用。
> **解决：** 增加"任务分析器"，自动判断并触发并行。

#### 3.1.1 任务分析器（新增强制规则）


**目标：** 每次执行任务前自动判断是否并行，无需用户提醒

**规则：** 在 `AGENTS.md` 新增以下规则：

```markdown
## 并行执行触发规则（强制）

**【强制执行】每次执行任务前：**


1. **分析任务**
   - 任务是否可以拆分为独立子任务？
   - 是否需要多个工具/文件/操作？
   - 是否可以同时执行不同方面的工作？

2. **判断标准**
   - ✅ 可以拆分 → **自动并行**，创建子Agent
   - ❌ 单一任务 → 串行执行

3. **并行阈值**
   - 2-3个独立子任务 → 并行执行
   - 4+个子任务 → 优先级排序后并行

4. **无需提醒**
   - **我主动判断，用户无需提醒**
   - 检测到可并行模式立即执行
```

#### 3.1.2 可并行模式判断


| 模式 | 判断标准 | 示例 |
|------|----------|------|
| **多文件操作** | 同时操作≥3个不同文件/目录 | "扫描E盘所有PDF" |
| **多工具调用** | 需要调用≥2个不相关工具 | "搜索文件+清理+备份" |
| **研究类任务** | 需要多角度分析 | "调研AI工具+整理报告" |
| **独立子任务** | 任务可拆分为相互独立的子任务 | "帮我做三件事" |
| **并行写作** | 多章节/多文档同时生成 | "写小说5-8章" |

#### 3.1.3 执行流程对比

**优化前（被动）：**
```
用户: "你可以并行执行吗？"
元神: 好 → 才创建子Agent
```

**优化后（主动）：**
```
用户: "帮我做这三件事"
元神: 分析 → 可并行 → 自动创建子Agent → 并行执行
```

#### 3.1.4 实现方式


在 AGENTS.md 添加：

```markdown
## 任务并行判断流程

**每次执行阳神任务前，必须问自己：**

1. 这个任务能不能拆成独立子任务？
2. 拆分的子任务是否相互独立（不需要等前一个完成）？
3. 是否有≥2个操作需要同时执行？

**如果任一为YES：**
- 自动调用 subagent-manager 创建子Agent
- 并行执行各子任务
- 汇总结果后汇报

**如果全部为NO：**
- 串行执行
- 正常调用阳神
```

## 十、新增优化项（基于深度反思 2026-05-12）

### 10.1 Cattle Policy（牛群策略）

**核心洞察：** Pet vs Cattle 思维转变
- 子Agent是"牛群"，不是"宠物"
- 失败后自动替换，不人工介入
- 文件：brain/cattle-policy.md

**处理规则：**
| 失败类型 | 处理方式 | 重试次数 |
|---------|---------|---------|
| 子Agent超时 | 终止 + 替换新实例 | 最多3次 |
| 子Agent崩溃 | 自动创建新实例继续 | 最多3次 |
| 模型API失败 | 等待 + 重试 + 换模型 | 最多3次 |

### 10.2 稳定接口设计

**核心洞察：** 元编排层（Meta-Harness）理念
- 最重要的不是"今天怎么做"，而是"未来怎么改"
- 定义稳定接口，实现可替换
- 文件：brain/interfaces.md

**核心接口：**
```typescript
interface ExecutionUnit {
  execute(task): Promise<Result>  // 执行任务
  health(): boolean                 // 健康检查
  replace(): void                   // 替换实例
}

interface TaskQueue {
  enqueue(task): taskId            // 入队
  dequeue(): Task | null           // 出队
  status(taskId): TaskStatus       // 状态查询
}

interface Storage {
  write(key, value): void         // 写入
  read(key): any                   // 读取
  query(filter): any[]             // 查询
}
```
