# 预热策略（Warm-up Strategy）

> 基于 Anthropic Managed Agents 的 TTFT 优化理念
> 目标：降低首token延迟，提升用户体验
> 生成时间：2026-05-12

## 背景

Anthropic 数据显示：
- 解耦后 **p50 TTFT 下降约 60%**
- 解耦后 **p95 TTFT 下降超过 90%**

TTFT = Time-to-First-Token（用户收到第一个token的时间）

## 当前问题（冷启动）

```
用户提问
    ↓
加载上下文（2-5秒）
    ↓
初始化工具（1-3秒）
    ↓
开始推理
    ↓
用户等待总计：3-8秒
```

## 优化后（热启动）

```
系统常驻组件（已预热）：
- 常用工具已初始化
- 子Agent池保持idle状态
- 上下文缓存已准备好

用户提问
    ↓
推理立即开始
    ↓
需要时调用预热的工具
    ↓
用户等待总计：<1秒
```

## 预热组件

### 常驻组件池

| 组件 | 预热状态 | 激活时机 | 说明 |
|------|---------|---------|------|
| 文件搜索Agent | idle | 系统启动时 | 快速响应搜索任务 |
| 日常工具层 | hot | 系统启动时 | 常用exec/文件操作 |
| 上下文缓存 | warm | 持续 | 最近对话保留 |

### 预热策略

```javascript
// 系统启动时执行
const warmup = async () => {
  // 1. 初始化Agent池
  await initAgentPool({
    searchAgent: { idle: true },
    fileAgent: { idle: true },
    commonTools: { loaded: true }
  })
  
  // 2. 预加载最近上下文
  await preloadRecentContext()
  
  // 3. 保持idle状态
  maintainIdlePool()
}
```

## 激活时机规则

### 触发预热
- 系统启动时
- 用户首次交互时
- 空闲时段定期预热

### 保持预热
- 任务完成后**不解散**，改为idle等待
- 空闲超时后才释放资源

### 释放预热
- 超过30分钟无任务
- 系统内存紧张时
- 用户明确要求时

## 预期效果

| 指标 | 优化前 | 优化后 |
|------|--------|---------|
| TTFT p50 | ~3秒 | ~1秒 |
| TTFT p95 | ~8秒 | ~2秒 |
| 用户感知 | "有点慢" | "立即响应" |

## 注意事项

- 预热会消耗一定内存（~50MB）
- 平衡：响应速度 vs 资源消耗
- 移动端/低内存设备慎用
