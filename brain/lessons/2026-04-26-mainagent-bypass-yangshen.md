# 教训：主Agent绕过阳神系统直接调用工具

**日期：** 2026-04-26
**类型：** 系统行为约束

## 问题描述

主Agent（我）在执行任务时，总是直接调用生成工具（music_generate / image_generate），跳过阳神系统。

### 违反记录
- **时间：** 2026-04-26 20:15
- **任务：** 生成艾薇儿风格歌曲 + 海报
- **行为：** 主Agent直接调用 music_generate + image_generate
- **问题：** 跳过了阳神系统的 orchestrator / 因果链 / 预防系统

## 根本原因

1. 主Agent默认行为是"自己干"，而不是"分配任务"
2. 没有系统级约束强制走阳神流程
3. 阳神系统存在但没有硬性拦截机制

## 解决方案（已实施）

### 1. SOUL.md 更新 - 核心身份明确

```markdown
**触发条件（满足任一即触发）：**
| 条件 | 说明 |
|------|------|
| 工具数 ≥2 | 任务需要多个工具并行或串行 |
| 生成类工具 | music_generate / image_generate / video_generate |
```

### 2. AGENTS.md 更新 - 工具调用规则扩展

```markdown
**触发条件（满足任一即触发）：**
- 工具数 ≥2
- 生成类工具（music/image/video）
- 并行需求

**执行流程：**
用户任务 → 主Agent判断 → 是 → 调用阳神系统
```

## 教训总结

| 教训 | 说明 |
|------|------|
| 系统存在 ≠ 会被使用 | 需要硬性约束才能保证执行 |
| 默认行为决定一切 | 必须改变"自己干"的默认习惯 |
| 规则要具体 | "复杂任务用阳神"不够，要明确"≥2工具" |

## 验证方法

下次执行任务时检查：
1. 主Agent是否先判断工具数量？
2. 是否调用了 orchestrator.executeTask()？
3. 子Agent是否实际执行了工具调用？
