# 教训：并行执行原则（2026-04-26）

## 错误

用户给两个任务：
1. 生成摇滚歌曲
2. 生成动漫海报

我的错误做法（串行）：
1. 先召唤歌词Agent
2. 等歌词完成
3. 再召唤海报Agent
4. 等海报设计完成
5. 再生成图片

用户质问："为什么没有并行"

## 正确做法（并行）

当有多个独立任务时，应该**同时召唤所有需要的Agent**：

```
并行调度：
├── 任务1：歌词创作
│   └── Agent: song-writer
│       ↓ 完成
│   └── music_generate (并行)
│
└── 任务2：海报设计
    └── Agent: poster-designer
        ↓ 完成
    └── image_generate (并行)
```

## 阳神系统核心原则

**并行 > 串行**
- 多个独立任务同时执行
- 不等待一个完成再启动下一个
- 全部完成后汇总结果

## 记忆

用户明确指出我没有并行执行，这是对阳神系统的误解。
阳神系统就是并行调度的系统，不是串行等待。
