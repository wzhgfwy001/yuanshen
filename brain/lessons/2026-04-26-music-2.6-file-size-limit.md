# 教训：music-2.6 文件大小限制问题

**日期：** 2026-04-26
**类型：** 技术配置 + 调试教训

## 问题描述

music_generate 调用 music-2.6 时，生成的音乐文件超过5MB限制导致失败。

**错误信息：** `Media exceeds 5MB limit`

## 根本原因

1. 生成的音乐时长太长 → 文件过大
2. MiniMax 对发送的文件有5MB硬性限制

## 解决方案

| 参数 | 值 | 说明 |
|------|-----|------|
| durationSeconds | ≤30秒 | 建议30秒以内 |
| model | minimax/music-2.6 | 用户指定模型 |

## 教训总结

### 1. action=list 不代表全部模型
- `music_generate action=list` 只显示部分模型（music-2.5+, music-2.5, music-2.0）
- music-2.6 不在列表中但实际可用
- **不要依赖 action=list 来判断模型是否存在**

### 2. 配置文件 vs 实际情况
- MEMORY.md 写的是 100次/日
- 实际应该是 100次/月
- 配置文件需要与用户后台实际显示一致

### 3. 生成工具调试流程
1. 先检查 action=list 可用模型
2. 但不要完全信任列表（可能不完整）
3. 实际调用测试
4. 注意文件大小限制

## 相关配置

- `package-support.json` - 已更新 music-2.6 默认模型
- `generation-tools-guard.js` - 逻辑正常
- `MEMORY.md` - 已更新配额信息

## 验证结果

✅ music-2.6 + durationSeconds=30 → 成功生成并发送
