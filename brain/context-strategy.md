# 上下文工程分层（Context Engineering）

> 基于 Anthropic Managed Agents 的 Session/Harness 分离理念
> 目标：存储层和策略层分离，让上下文管理更灵活
> 生成时间：2026-05-12

## 核心洞察

**Anthropic观点：**
> "Session 负责存储，Harness 负责上下文工程策略"
> "两者分开，不要耦合"

## 当前问题（耦合）

```
brain/ (存储)
    ↓
直接读进AI上下文
    ↓
AI自己决定怎么用这些信息
    ↓
策略硬编码在AI的prompt里
```

**问题：**
- 如果要换上下文策略（比如从压缩改外置记忆）→ 要改AI整个系统
- 上下文满了只能压缩/裁剪，无法扩展

## 优化后架构（解耦）

```
┌─────────────────────────────────────┐
│        AI 上下文（有限窗口）           │
└─────────────────────────────────────┘
                    ↑
              ┌─────────────┐
              │ Context    │  ← 策略层
              │ Engine     │
              └─────────────┘
                    ↑ 管理
              ┌─────────────┐
              │   brain/   │  ← 存储层
              │ (外部无限) │
              └─────────────┘
```

## 两层职责

### 存储层（brain/）

| 职责 | 说明 |
|------|------|
| 持久化 | 所有事件和记忆永久保存 |
| 可查询 | 支持按时间/类型/标签筛选 |
| 可恢复 | 崩溃后可以从这里恢复 |

### 策略层（Context Engine）

| 职责 | 说明 |
|------|------|
| 决定给AI什么 | 从brain/取什么进上下文 |
| 组织方式 | 如何组织、排序、压缩 |
| 适配模型 | 根据模型能力调整策略 |

## 上下文工程策略

### 策略1：Compaction（压缩）

```
原始对话 (10,000 tokens)
        ↓
    LLM摘要
        ↓
压缩后 (500 tokens)
```

**适用场景：** 长对话，需要跨会话推理

### 策略2：External Memory（外置记忆）

```
上下文窗口（工作记忆）
        ↑  只加载相关片段
brain/（永久记忆）
```

**适用场景：** 需要引用很久以前的记忆

### 策略3：Semantic Chunking（语义分块）

```
原始文本
    ↓ 按语义分割
[块1: 概念A] [块2: 概念B] [块3: 概念C]
    ↓ 按需加载
AI上下文
```

**适用场景：** 大量文档检索

### 策略4：Hybrid（混合）

```
最近对话 → 直接加载（保持新鲜）
旧对话 → 摘要后加载（节省空间）
重要记忆 → 完整保留（关键信息）
```

**适用场景：大多数场景，推荐使用**

## 我们的实现

### 当前（简化版）

```markdown
## 简化策略

1. 最近对话 → 全部保留
2. 旧记忆 → 标签聚合
3. 工具结果 → 用完即弃（不保留）

未来可以升级为：
- [ ] Compaction
- [ ] External Memory
- [ ] Semantic Chunking
```

### 升级路径

```
当前 → Hybrid混合 → 未来可演进到任意策略
         ↓
    策略可配置
         ↓
    底层存储不变
```

## 好处

| 好处 | 说明 |
|------|------|
| 策略可换 | 压缩改外置记忆 → 只需改Context Engine |
| 性能提升 | 按需加载，不用全部塞进上下文 |
| 可调试 | 策略层可以单独测试 |
| 未来适配 | 新模型来了换策略层就行 |
