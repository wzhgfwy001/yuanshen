# StoryFlow 用户故事

## 概述

本文档通过用户故事描述不同场景下如何使用 StoryFlow，帮助用户理解产品价值和使用方式。

---

## 故事 1：新手小明的第一个工作流

### 背景设置

**人物：** 小明
**身份：** 内容创作者，每周需要发布 2-3 篇博客文章
**痛点：**
- 每次写作都要花很多时间构思创意
- 写作流程不系统，效率低下
- 想尝试 AI 辅助但不知道如何入手
- 对编程不熟悉，担心技术门槛高

### 用户旅程

#### 第一步：发现产品

小明在一个技术博客上看到了 StoryFlow 的介绍文章：

> "StoryFlow 是一个无需编程的 AI 工作流工具，通过拖拽节点就能构建复杂的内容生成流程。就像 ComfyUI 一样直观，但专注于文本内容创作。"

他被这句话吸引了，决定试试看。

#### 第二步：安装和首次启动

**CLI 版本尝试：**

```bash
# 小明按照教程安装
npm install -g @storyflow/cli

# 初始化项目
storyflow init

# 看到了友好的欢迎信息
✓ StoryFlow 配置文件已创建
✓ 工作流目录已初始化
✓ 示例工作流已添加

下一步：
  - 编辑 storyflow.config.yaml 配置 API keys
  - 运行示例: storyflow run examples/hello-world.yaml
```

**发现 CLI 对他来说太技术化了，决定尝试 Web 版本。**

#### 第三步：Web 版本初体验

1. 打开 https://storyflow.ai
2. 注册账号（使用 Google 账号登录）
3. 看到**新手引导弹窗**：

```
┌──────────────────────────────────────┐
│ 👋 欢迎使用 StoryFlow！               │
├──────────────────────────────────────┤
│ 让我们花 2 分钟了解基本操作：         │
│                                      │
│ 1️⃣ 从左侧拖拽节点到画布              │
│ 2️⃣ 点击端口创建连线                  │
│ 3️⃣ 点击"运行"按钮执行工作流          │
│                                      │
│ [开始教程]  [跳过，直接探索]          │
└──────────────────────────────────────┘
```

小明选择"开始教程"。

#### 第四步：完成交互式教程

**教程步骤 1：添加第一个节点**

```
🎯 任务：从左侧节点库拖拽一个"LLM"节点到画布中央

提示：你可以搜索"LLM"或直接在"LLM"分类下找到
```

小明照做了，成功添加了节点。

**教程步骤 2：配置节点**

```
🎯 任务：点击节点，在属性面板中输入提示词

提示：输入"写一段关于春天的诗"
```

小明在提示词框中输入了文字。

**教程步骤 3：运行工作流**

```
🎯 任务：点击顶部的"运行"按钮

提示：观察节点的状态变化
```

点击后，节点边框开始闪烁，几秒后变成绿色，并输出了结果：

```
春光洒满大地间，
万物复苏展欢颜。
花开叶绿生机现，
岁月静好暖心田。
```

"太神奇了！"小明兴奋地说。

**教程步骤 4：保存工作流**

```
🎯 任务：按 Ctrl+S (或 Cmd+S) 保存工作流

提示：给工作流起个名字吧
```

小明输入："我的第一个工作流"，点击保存。

```
✅ 恭喜！你已经完成了第一个工作流！

下一步：
  • 查看模板库，探索更多可能性
  • 学习如何添加多个节点构建复杂流程
  • 加入社区，分享你的作品

[去模板库]  [查看文档]  [关闭]
```

#### 第五步：创建博客生成工作流

小明想创建一个真正能用的博客生成器。他：

1. 打开模板库
2. 搜索"博客"
3. 找到"博客文章生成器"模板

**模板详情页：**

```
📝 博客文章生成器

描述:
从主题自动生成完整的博客文章，包含创意生成、内容选择、扩展和格式化四个步骤。

特性:
✓ 自动生成 5 个创意
✓ 智能选择最佳创意
✓ 支持自定义文章长度
✓ 可调整写作风格

使用次数: 1,234  ⭐ 4.8  👍 156

[使用此模板] [查看详情]
```

小明点击"使用此模板"。

#### 第六步：定制工作流

模板在画布中打开，包含 4 个节点：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  生成创意   │───▶│  选择最佳   │───▶│  扩展内容   │───▶│  格式化输出 │
│   (LLM)     │    │   (LLM)     │    │   (LLM)     │    │   (格式化)   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

小明开始定制：

1. **修改"生成创意"节点：**
   - 将主题改为"AI 工作流自动化"
   - 保留默认的 5 个创意数量

2. **修改"扩展内容"节点：**
   - 调整温度参数从 0.7 到 0.8（更有创意）
   - 修改提示词，强调实用性和可操作性

3. **测试节点：**
   - 点击"生成创意"节点的"测试"按钮
   - 看到了 5 个不错的创意：
     ```
     1. "AI 工作流自动化的 5 个实用技巧"
     2. "如何用 AI 将你的工作效率提升 10 倍"
     3. "从零开始构建你的第一个 AI 工作流"
     4. "AI 工作流 vs 传统自动化：差异与选择"
     5. "2024 年最值得关注的 AI 工作流工具"
     ```

4. **运行完整工作流：**
   - 点击"运行"按钮
   - 观察每个节点的执行进度
   - 最终获得一篇完整的博客文章（1200 字）

#### 第七步：保存和导出

小明对结果很满意：

1. **保存工作流：**
   ```yaml
   保存位置: 我的工作流
   名称: AI 工作流博客生成器
   标签: 博客, AI, 自动化
   ```

2. **导出结果：**
   - 选择 Markdown 格式
   - 下载到本地
   - 直接复制到他的博客平台

#### 第八步：优化迭代

一周后，小明发现：

- 生成的文章结构很好，但有时过于正式
- 想加入更多个人风格

他回到工作流，进行优化：

1. 在"扩展内容"节点前添加一个"风格设置"节点
2. 输入自己的写作风格描述：
   ```
   你的风格：
   - 用生动的例子和比喻
   - 加入个人经历和见解
   - 语言轻松但不失专业
   - 偶尔使用幽默
   ```

3. 重新运行，结果更符合他的风格！

#### 用户反馈

小明在使用两周后的评价：

> "StoryFlow 完全改变了我写博客的方式！
>
> **优点：**
> • 不需要编程，可视化界面非常直观
> • 模板库让我快速上手，不用从零开始
> • 可以逐步优化工作流，每次运行都有进步
> • 节省了 70% 的写作时间
>
> **遇到的小问题：**
> • 最初不知道如何调试失败的节点（后来在文档中找到了答案）
> • 想添加自定义节点但不太容易（发现可以通过插件实现）
>
> **建议：**
> • 希望有更多行业特定的模板
> • 希望能直接导出到我的博客平台
>
> **总体评分：⭐⭐⭐⭐⭐**

## 故事 2：产品经理小红的工作流优化

### 背景设置

**人物：** 小红
**身份：** SaaS 产品经理，负责产品文档和用户指南
**痛点：**
- 产品更新频繁，文档总是跟不上
- 每次更新都要手动同步多处文档
- 想要统一的文档生成流程
- 团队协作需要标准化

### 用户旅程

#### 第一步：分析需求

小红列出需求：

1. 从更新日志自动生成：
   - 用户指南更新
   - API 文档更新
   - 帮助中心文章
   - 产品公告邮件

2. 输入：
   - 产品更新日志（JSON 格式）
   - 产品名称和版本
   - 目标受众（开发者/用户/管理员）

3. 输出：
   - 格式化的 Markdown 文档
   - HTML 预览
   - 邮件草稿

#### 第二步：构建工作流

小红创建了一个名为"文档自动化生成器"的工作流：

**节点 1：读取更新日志**

```
类型: 文件读取
输入: updates.json
输出: raw_updates
```

**节点 2：解析更新内容**

```
类型: JSON 解析
输入: {{raw_updates}}
配置:
  - 提取版本号
  - 按类型分类（新功能/修复/改进）
  - 识别影响范围
输出: parsed_updates
```

**节点 3：生成用户指南**

```
类型: LLM
输入: {{parsed_updates}}
提示词:
  基于以下产品更新，生成用户指南更新部分：

  更新内容：{{parsed_updates}}

  要求：
  1. 突出对用户最有价值的新功能
  2. 用简单的语言解释技术变化
  3. 提供使用示例
  4. 包含"如何开始"部分
  5. 格式：Markdown
输出: user_guide_updates
```

**节点 4：生成 API 文档**

```
类型: LLM
输入: {{parsed_updates}}
提示词:
  为开发者生成 API 更新文档：

  更新内容：{{parsed_updates}}

  要求：
  1. 列出所有 API 变更（新增/修改/废弃）
  2. 提供代码示例
  3. 标注兼容性影响
  4. 包含迁移指南
  5. 格式：Markdown
output: api_docs_updates
```

**节点 5：生成产品公告**

```
类型: LLM
输入: {{parsed_updates}}
提示词:
  写一封产品公告邮件：

  更新内容：{{parsed_updates}}
  受众: 所有用户
  语调: 专业但友好

  要求：
  1. 引人入胜的标题
  2. 简短引言
  3. 重点功能介绍（最多 3 个）
  4. 升级指引
  5. 行动号召
output: announcement_email
```

**节点 6：格式化用户指南**

```
类型: 格式化工具
输入: {{user_guide_updates}}
配置:
  - 添加目录
  - 统一标题层级
  - 格式化代码块
output: formatted_user_guide
```

**节点 7：格式化 API 文档**

```
类型: 格式化工具
input: {{api_docs_updates}}
配置:
  - 生成 API 变更对照表
  - 添加废弃警告
  - 生成示例代码
output: formatted_api_docs
```

**节点 8：保存输出**

```
类型: 文件保存
输入:
  - {{formatted_user_guide}}
  - {{formatted_api_docs}}
  - {{announcement_email}}
路径: outputs/docs-{{timestamp}}/
```

**完整工作流图：**

```
┌─────────────┐
│  读取日志   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  解析更新   │
└──────┬──────┘
       │
   ┌───┴───┬────────┐
   ▼       ▼        ▼
┌──────┐ ┌──────┐ ┌──────┐
│用户指│ │API   │ │公告  │
│南节点│ │文档  │ │邮件  │
└──┬───┘ └──┬───┘ └──┬───┘
   │        │        │
   ▼        ▼        ▼
┌──────┐ ┌──────┐    │
│格式化│ │格式化│    │
└──┬───┘ └──┬───┘    │
   │        │        │
   └────────┴────────┘
            │
            ▼
       ┌─────────┐
       │ 保存输出│
       └─────────┘
```

#### 第三步：测试和调试

小红第一次运行工作流，遇到了问题：

```
✗ 节点"解析更新"执行失败
错误: Invalid JSON format

原始数据:
{
  "version": "2.5.0",
  "updates": [
    {"type": "feature", "title": "..."}
```

问题：updates.json 格式不规范。

**解决方案：**

小红在"解析更新"节点前添加了一个"数据清理"节点：

```
类型: 数据清洗
输入: {{raw_updates}}
配置:
  - 修复 JSON 格式
  - 去除注释
  - 标准化字段名
output: cleaned_updates
```

重新运行，成功！

#### 第四步：团队协作

小红将工作流分享给团队：

1. **生成分享链接：**
   ```
   🔗 https://storyflow.ai/s/doc-gen-xyz123
   权限: 可编辑
   ```

2. **团队反馈：**
   - 技术文档工程师："API 文档格式需要调整"
   - UX 设计师："用户指南需要更多截图说明"

3. **协作优化：**
   - 小红邀请团队成员共同编辑
   - 每个人负责优化自己相关节点的提示词
   - 版本历史记录所有修改

#### 第五步：自动化部署

小红将工作流集成到 CI/CD 流程：

```bash
# 在 CI 脚本中添加
echo "开始生成文档..."
storyflow run workflows/doc-generator.yaml \
  --env .env.production \
  --save-result outputs/docs/ \
  --output-format json

echo "文档生成完成，开始部署..."
rsync -avz outputs/docs/ /var/www/docs/

echo "文档已更新！"
```

每次产品发布时自动运行，无需手动干预。

#### 第六步：监控和优化

小红设置监控：

1. **执行统计：**
   - 平均执行时间：45 秒
   - 成功率：98%
   - API 成本：每次 $0.15

2. **发现优化点：**
   - "格式化"节点可以并行执行
   - 用户指南和 API 文档生成可以同时进行

3. **性能优化：**
   - 将并行节点设置并发执行
   - 添加缓存（未变更的内容不重新生成）
   - 执行时间减少到 30 秒

#### 用户反馈

小红的使用体验：

> "StoryFlow 让我们的文档工作流程完全自动化了！
>
> **效果：**
> • 文档更新时间从 2 小时缩短到 1 分钟
> • 文档一致性显著提高
> • 团队协作效率提升 3 倍
>
> **价值：**
> • 产品发布当天就能看到完整文档
> • 文档质量标准化
> • 新人可以快速上手文档工作
>
> **印象深刻的功能：**
> • 版本历史和对比功能非常实用
> • 节点测试让调试变得简单
> • CI/CD 集成无缝衔接
>
> **改进建议：**
> • 希望有更丰富的文档模板
> • 需要更细粒度的权限控制（某些人只能编辑特定节点）"

## 故事 3：自由职业者小张的提示词调试

### 背景设置

**人物：** 小张
**身份：** AI 提示词工程师，为客户提供 AI 文案服务
**痛点：**
- 提示词调试过程繁琐，需要反复试错
- 客户需求变化，需要快速调整
- 想要积累和复用成功的提示词模式
- 需要向客户展示工作流程

### 用户旅程

#### 第一步：接到客户需求

客户需求：

> "帮我们创建一个产品描述生成器，需要：
> 1. 根据产品特性生成吸引人的描述
> 2. 支持 3 种语调（专业/轻松/促销）
> 3. 长度可调（短/中/长）
> 4. 包含 SEO 关键词"

#### 第二步：设计工作流

小张开始设计：

```
输入层:
  - 产品特性（文本）
  - 目标语调（选择）
  - 描述长度（选择）
  - SEO 关键词（文本）

处理层:
  1. 分析产品特性
  2. 根据语调调整风格
  3. 根据长度控制输出
  4. 融入 SEO 关键词
  5. 优化可读性

输出层:
  - 产品描述
  - 变体 A（备选）
  - 变体 B（备选）
  - SEO 得分
```

#### 第三步：实现工作流

**节点 1：输入验证**

```
类型: 条件分支
输入: product_features

检查:
  - 产品特性不能为空
  - 关键词至少 3 个
  - 语调选择有效

输出: validation_result
```

**节点 2：产品分析**

```
类型: LLM
输入: {{product_features}}
提示词:
  分析以下产品特性，提取关键信息：

  产品：{{product_features}}

  输出格式（JSON）：
  {
    "core_value": "核心价值主张",
    "target_audience": "目标用户",
    "unique_selling_points": ["独特卖点1", "卖点2"],
    "key_benefits": ["利益1", "利益2"],
    "use_cases": ["使用场景1", "场景2"]
  }
output: product_analysis
```

**节点 3：语调适配**

```
类型: LLM
输入:
  - {{product_analysis}}
  - {{tone}}
提示词:
  根据{{tone}}语调调整产品分析：

  产品分析：{{product_analysis}}
  语调：{{tone}}

  语调指南：
  - 专业：正式、权威、数据驱动
  - 轻松：友好、对话式、故事化
  - 促销：紧迫、行动导向、利益突出

  输出调整后的产品分析。
output: toned_analysis
```

**节点 4：生成描述（主版本）**

```
类型: LLM
输入:
  - {{toned_analysis}}
  - {{length}}
  - {{seo_keywords}}
提示词:
  基于以下信息生成产品描述：

  产品分析：{{toned_analysis}}
  长度：{{length}}（短:100字/中:300字/长:600字）
  关键词：{{seo_keywords}}

  要求：
  1. 吸引人的标题
  2. 突出核心价值
  3. 自然融入关键词
  4. 强有力的行动号召
  5. 语调符合{{tone}}风格
output: main_description
```

**节点 5：生成变体 A**

```
类型: LLM
input: {{main_description}}
提示词:
  基于以下描述，创建一个略有不同的变体：

  原描述：{{main_description}}

  变化策略：
  - 使用不同的开头
  - 调整结构
  - 改变行动号召方式

  保持核心信息和语调一致。
output: variant_a
```

**节点 6：生成变体 B**

```
类型: LLM
input: {{main_description}}
提示词:
  创建第三个描述变体，采用不同的角度：

  原描述：{{main_description}}

  新角度：
  - 以用户痛点开头
  - 使用对比手法
  - 加入社会证明元素

output: variant_b
```

**节点 7：SEO 评分**

```
类型: SEO 分析工具
input:
  - {{main_description}}
  - {{seo_keywords}}
分析:
  - 关键词密度
  - 关键词位置
  - 标题优化
  - 可读性得分
output: seo_score
```
