# StoryFlow 原型描述

## 概述

本文档描述 StoryFlow 的界面原型，使用 ASCII 艺术展示核心界面的布局和交互。

---

## 1. CLI 启动流程

```bash

运行以下命令初始化：
  storyflow init
```

### 1.4 错误处理

```bash
$ storyflow run broken-workflow.yaml

✗ 配置文件格式错误
  文件: broken-workflow.yaml
  位置: 第 12 行，第 5 列

问题:
  缺少必需字段 'id' in nodes[1]

  当前配置:
  nodes:
    - id: node_1
      name: "第一个节点"
    - name: "第二个节点"  ← 这里缺少 id
      type: llm

修复建议:
  1. 为节点添加 id 字段
  2. 运行 storyflow validate 先验证
  3. 参考 examples/ 中的示例

提示：YAML 对缩进敏感，确保使用 2 个空格缩进

详细错误信息:
  ValidationError: Missing required field: 'id'
    at /nodes/1
    in broken-workflow.yaml:12:5

使用 --verbose 查看完整堆栈跟踪
```

### 1.5 交互式帮助

```bash
$ storyflow

StoryFlow v1.0.0 - AI 工作流引擎

使用方法:
  storyflow <command> [options]

可用命令:
  📁 init          初始化 StoryFlow 项目
  ▶️  run           运行工作流
  ✓   validate      验证工作流文件
  📋 list          列出可用工作流
  📦 template      管理工作流模板
  ⚙️  config        管理配置
  📖 docs          打开文档

全局选项:
  -h, --help       显示帮助信息
  -v, --version    显示版本信息
  --debug          启用调试模式

获取命令详细帮助:
  storyflow <command> --help

示例:
  storyflow run workflow.yaml --verbose
  storyflow init --template advanced
  storyflow validate my-workflow.yaml

快速开始:
  1. storyflow init              # 初始化项目
  2. storyflow run examples/*    # 运行示例
  3. storyflow docs              # 查看文档
```

```
  │ 节点内容      │  (白色文字)
  └──────────────┘  (白色边框)
```

## 总结

这些原型描述展示了 StoryFlow 的核心界面和交互设计：

1. **CLI 界面**：简洁、高效、易于集成
2. **Web 界面**：直观、强大、可视化
3. **响应式设计**：适配不同设备
4. **交互反馈**：清晰、友好、及时
5. **设计规范**：一致、专业、可扩展

设计核心原则：
- 简单易用：降低学习成本
- 视觉清晰：明确的信息层级
- 响应迅速：流畅的交互体验
- 可访问性：支持所有用户
