# 使用情况统计说明

**用途：** 了解用户使用习惯，持续优化系统
**原则：** 自愿参与，匿名收集，仅用于改进

---

## 📊 收集内容

### 基础数据（自动）
- 任务类型（单一/标准/创新/复杂）
- 执行时长
- 子 Agent 数量
- Token 消耗量

### 可选数据（手动填写）
- 满意度评分
- 使用场景
- 改进建议

---

## 🔒 隐私保护

- ✅ **匿名收集** - 不收集个人信息
- ✅ **本地存储** - 数据存储在本地
- ✅ **自愿发送** - 用户选择是否分享
- ✅ **仅用于改进** - 不用于商业用途

---

## 📝 参与方式

### 方式 1：手动填写（推荐）

复制以下模板，填写后发送到 ClawHub 评论区：

```markdown
## 使用统计

**使用日期：** YYYY-MM-DD
**任务类型：** 单一/标准/创新/复杂
**执行时长：** ~X 秒
**子 Agent 数：** X 个
**Token 消耗：** ~X tokens

### 满意度
- ⭐⭐⭐⭐⭐ 非常满意
- ⭐⭐⭐⭐ 满意
- ⭐⭐⭐ 一般
- ⭐⭐ 不满意
- ⭐ 非常不满意

### 使用场景


### 改进建议


```

### 方式 2：自动统计（开发中）

未来版本将支持可选的自动统计功能：
- 首次运行时询问是否启用
- 数据本地加密存储
- 每周/月汇总发送

---

## 📈 数据用途

收集的数据将用于：
1. **性能优化** - 识别高 Token 消耗场景
2. **功能改进** - 根据使用场景调整功能
3. **问题修复** - 快速定位常见问题
4. **版本规划** - 数据驱动迭代方向
