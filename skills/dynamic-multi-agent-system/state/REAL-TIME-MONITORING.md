# 实时监控说明

## 📊 当前 Agent 管理机制

### ✅ 正确的机制（实际运行）
1. **按需创建** - 任务需要时动态生成子 Agent
2. **任务驱动** - 每个子 Agent 有明确任务
3. **完成后销毁** - 任务结束即清理资源
4. **无常驻 Agent** - 不保持空闲 Agent

### 📈 实时状态检查方法

#### 方法 1：检查 OpenClaw 会话
```bash
# 查看当前所有会话
openclaw sessions list

# 查看子 Agent 会话
openclaw sessions list --kind subagent
```

#### 方法 2：使用 sessions_list API
```json
{
  "kinds": ["agent:main:subagent"],
  "limit": 20
}
```

#### 方法 3：检查会话状态
- `status: "done"` - 已完成，已清理
- `status: "running"` - 正在运行
- `status: "failed"` - 失败，需要处理

**真正的 Token 消耗机制：**
1. **任务执行时** - 创建子 Agent，消耗 Token
2. **任务完成后** - 子 Agent 销毁，停止消耗
3. **无空闲消耗** - 没有常驻 Agent 消耗 Token
