## 📘 快速开始

### 什么是混合动态多 Agent 系统？

这是一个运行在 OpenClaw 之上的智能协作系统，能够：
- 🤖 **自动分解复杂任务** - 无需手动指定步骤
- 👥 **动态组建 Agent 团队** - 按需创建专业角色
- ✅ **三层质量保障** - 自我检查 + 主 Agent 确认 + 独立审查
- 📈 **持续进化学习** - 成功经验自动沉淀为可复用 Skill

---

## 🚀 基本用法

### 直接使用（推荐新手）

**无需特殊命令，直接描述任务即可：**

```
写一篇科幻短篇小说，主题是 2077 年的北京，要求温暖治愈风格
```

系统会自动：
1. 识别任务类型
2. 分解为子任务
3. 创建专业 Agent 团队
4. 协调执行并交付成果

---

### 进阶用法（指定要求）

**明确任务类型：**
```
这是一个创新任务，需要设计一套完整的公司管理体系
```

**指定质量要求：**
```
要求输出质量达到专业级别，需要详细的审查环节
```

**指定角色偏好：**
```
希望有行业专家参与，特别是科技公司的管理经验丰富的
```

## 📋 任务类型说明

系统支持 4 种任务类型，了解类型有助于获得更好结果：

### 1️⃣ 单一任务（Simple）

**特征：** 简单、快速、无需多步骤

**示例：**
- "翻译这段文字为英文"
- "总结这篇文章的核心观点"
- "计算这组数据的平均值"

**处理方式：** 主 Agent 直接执行或创建 1 个子 Agent

**预计耗时：** <5 分钟

---

### 2️⃣ 标准任务（Standard）

**特征：** 常见任务类型，有成熟流程

**示例：**
- "写一个科幻小说"
- "分析销售数据并生成报告"
- "开发一个待办事项应用"

**处理方式：** 使用固化的 Skill 流程执行

**预计耗时：** 60-90 分钟

**典型流程：**
```
任务分类 → 加载 Skill → 执行标准流程 → 质量检查 → 交付
```

---

### 3️⃣ 创新任务（Innovative）

**特征：** 复杂、新颖、无标准流程

**示例：**
- "设计一套完整的公司管理体系"
- "写一个推理小说，采用叙事诡计 + 时间诡计"
- "策划一场线上线下结合的营销活动"

**处理方式：** 动态组建 Agent 团队

**预计耗时：** 120-180 分钟

**典型流程：**
```
任务分类 → 动态分解 → 创建 Agent 团队 → 协调执行 → 三层审查 → 交付
```

---

### 4️⃣ 混合任务（Hybrid）

**特征：** 标准任务 + 定制要求

**示例：**
- "写一本科幻小说，重点描写人工智能伦理"
- "分析数据，但要对比行业基准"
- "开发一个应用，要求支持离线模式"

**处理方式：** 加载 Skill + 添加定制环节

**预计耗时：** 90-120 分钟

**典型流程：**
```
任务分类 → 加载 Skill → 添加定制角色 → 执行 → 定制审查 → 交付
```

## 💡 最佳实践

### 1. 清晰描述任务

**❌ 不好的描述：**
```
写个小说
```

**✅ 好的描述：**
```
创作一部悬疑推理短篇小说，采用暴风雪山庄模式，包含一个密室杀人案，共 10 章，每章约 1000 字
```

**为什么：** 清晰的描述帮助系统准确分类和分解

---

### 2. 明确质量要求

**❌ 不好的描述：**
```
写好一点
```

**✅ 好的描述：**
```
要求输出质量达到专业发表级别，需要严格的审查环节，质量分不低于 90 分
```

**为什么：** 明确的质量标准让审查 Agent 有据可依

---

### 3. 提供参考样例

**❌ 不好的描述：**
```
类似那种科幻小说
```

**✅ 好的描述：**
```
风格类似《三体》，硬科幻，注重科学细节，但基调更温暖治愈
```

**为什么：** 参考样例帮助 Agent 理解期望风格

---

### 4. 分阶段验收

对于大型任务，建议：
```
先输出大纲，我确认后再继续创作正文
```

**为什么：** 阶段性验收避免方向偏差

---

### 5. 善用反馈机制

任务完成后：
- ⭐⭐⭐⭐⭐ 超出预期 → 系统会计入 Skill 固化
- ⭐⭐⭐⭐ 满意 → 正常计入
- ⭐⭐⭐ 一般 → 记录问题
- ⭐⭐ 不满意 → 触发改进流程
- ⭐ 非常不满意 → 人工介入

**为什么：** 反馈帮助系统持续改进

## 🔍 查看进度

### 查询任务状态

```bash
# 查看当前运行的任务
openclaw tasks list

# 查看特定任务详情
openclaw tasks status <task-id>
```

### 查看子 Agent 状态

```bash
# 查看活跃的子 Agent
openclaw subagents list

# 查看子 Agent 详情
openclaw subagents describe <session-id>
```

## ⚠️ 常见问题

### Q1: 为什么任务耗时这么长？

**A:** 复杂任务需要多 Agent 协作，每个环节都需要时间：
- 世界观设计：15-20 分钟
- 大纲规划：10-15 分钟
- 内容创作：30-60 分钟
- 质量审查：10-20 分钟

**建议：** 对于紧急任务，明确说明"加急"，系统会优化流程

---

### Q2: 可以中途修改要求吗？

**A:** 可以，但可能影响进度：
- 小修改（如调整风格）：直接提出
- 大修改（如改变主题）：建议重新开始任务

**建议：** 在大纲阶段确认需求，避免后期大幅修改

---

### Q3: 如何确保输出质量？

**A:** 系统有三层质量保障：
1. 子 Agent 自我检查
2. 主 Agent 确认
3. 独立审查 Agent（复杂任务启用）

**建议：** 明确要求"启用审查 Agent"，质量更有保障

---

### Q4: 可以指定特定的 Agent 角色吗？

**A:** 可以，在任务描述中说明：
```
希望有心理学专家参与，分析人物动机
```

**建议：** 指定角色会让系统更精准地组建团队

---

### Q5: 任务失败怎么办？

**A:** 系统会自动重试（最多 3 次），如仍失败：
1. 检查任务描述是否清晰
2. 降低任务复杂度
3. 联系技术支持

**建议：** 分解为多个小任务分别执行

## 🎯 高级技巧

### 技巧 1：利用 Skill 固化

如果你经常执行同类任务（如每周数据报告），系统会在 3 次成功后自动固化流程，后续执行更快更稳定。

**提示：** 保持任务类型一致性，加速 Skill 固化

---

### 技巧 2：并行任务优化

系统支持最多 3 个主任务并行，可以：
```
同时提交 3 个独立任务，系统会自动调度
```

**注意：** 总子 Agent 数不超过 12 个

---

### 技巧 3：自定义审查清单

对于特定类型任务，可以提供审查清单：
```
审查重点：
- 数据准确性
- 逻辑连贯性
- 格式规范性
```

**好处：** 审查更有针对性

---

### 技巧 4：版本对比

如需对比多个版本：
```
请生成 3 个不同风格的版本，供我选择
```

**注意：** 会增加执行时间和资源使用
