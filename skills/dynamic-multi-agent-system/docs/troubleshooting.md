# 故障排除

## 常见问题

### 1. 任务分类错误

**症状：**
- 简单任务被分解为多个子 Agent
- 或者复杂任务没有被正确分解

**可能原因：**
- 任务描述模糊
- 置信度阈值设置过高

**解决方案：**
```
1. 明确任务描述，添加更多细节
2. 调整配置：confidence-threshold 降至 0.6
3. 手动指定："这是一个创新任务，需要分解为以下步骤..."
```

---

### 2. 子 Agent 创建失败

**症状：**
```
错误：无法创建子 Agent
```

**可能原因：**
- 达到最大子 Agent 数量限制（默认 12）
- 模型 API 限流
- 内存不足

**解决方案：**
```
1. 等待其他任务完成
2. 取消不必要的任务
3. 检查模型配额
4. 增加 max-sub-agents 配置（谨慎）
```

---

### 3. 任务执行超时

**症状：**
```
错误：子 Agent 执行超时（超过 5 分钟）
```

**可能原因：**
- 任务过于复杂
- 模型响应慢
- 网络问题

**解决方案：**
```
1. 分解为更小的子任务
2. 减少单次输出要求（如"先写前 3 章"）
3. 检查网络连接
4. 更换更快的模型
```

---

### 4. 质量检查不通过

**症状：**
```
质量检查未通过：
- 问题 1：逻辑不一致
- 问题 2：数据错误
```

**解决方案：**
```
选项 1：让系统自动重试（默认）
选项 2：手动修改要求后重新执行
选项 3：接受当前结果，标注已知问题
```

**命令：**
```
接受当前结果，记录问题：[问题描述]
```

---

### 5. Skill 固化失败

**症状：**
- 任务成功 3 次以上，但没有固化候选

**可能原因：**
- enable-evolution 配置为 false
- 成功率低于 0.8
- 流程不一致（每次执行差异太大）

**解决方案：**
```
1. 检查配置：enable-evolution: true
2. 确保任务流程一致
3. 手动固化：固化 [流程名] 为 Skill
```

---

### 6. 资源清理不彻底

**症状：**
- 磁盘空间持续增长
- state/execution-logs/ 文件过多

**解决方案：**
```
1. 手动清理：清理临时资源
2. 调整保留期限：清理 7 天前的日志
3. 禁用详细日志（如不需要调试）
```

---

### 7. 多任务并发冲突

**症状：**
- 任务 A 的输出影响了任务 B
- 上下文混乱

**可能原因：**
- 共享记忆层隔离不当
- 子 Agent 使用了错误的上下文

**解决方案：**
```
1. 减少并发任务数：max-concurrent-tasks: 1
2. 等待一个任务完成再开始下一个
3. 检查任务描述是否清晰独立
```

## 日志分析

### 日志位置

```
state/execution-logs/
├── 2026-04-04-task-005.json
├── 2026-04-03-task-004.json
└── ...
```

### 分析步骤

1. **找到失败任务的日志文件**
2. **检查失败阶段**
   ```json
   "phases": [
     {"phase": "任务分类", "result": "success"},
     {"phase": "任务分解", "result": "success"},
     {"phase": "子 Agent 执行", "result": "failed", "error": "..."}
   ]
   ```
3. **查看错误详情**
   ```json
   "error-details": {
     "agent": "小说写作专家",
     "error": "超时",
     "duration": 320
   }
   ```
4. **根据错误类型采取对应措施**

## 性能优化

### 减少执行时间

1. **减少子 Agent 数量**
   - 合并相关职责
   - 例如：将"大纲设计"和"章节写作"合并

2. **增加并行度**
   - 识别可并行的子任务
   - 调整依赖关系

3. **简化输出要求**
   - 分批交付（先交付大纲，再交付内容）
   - 减少单次输出长度

### 减少资源消耗

1. **降低并发任务数**
   ```json
   "max-concurrent-tasks": 1
   ```

2. **禁用详细日志**（非调试时）

3. **定期清理旧日志**

## 紧急处理

### 系统卡死

**症状：** 长时间无响应（>10 分钟）

**处理：**
```
1. 发送：取消当前任务
2. 如无效，重启 OpenClaw Gateway
3. 检查日志定位原因
```

### 数据损坏

**症状：** 状态文件无法读取

**处理：**
```
1. 从备份恢复 state/ 目录
2. 如无备份，删除损坏文件（会丢失统计）
3. 系统会自动重建空状态文件
```
