## 📝 示例 1：科幻创作（标准任务）

### 任务描述

```
写一篇科幻短篇小说
主题：2077 年的北京清晨
要求：
- 800-1000 字
- 有明确的世界观设定
- 体现科技与生活的融合
- 温暖治愈风格
```

### 执行流程

**1. 任务分类**
```json
{
  "type": "standard",
  "confidence": 0.85,
  "reasoning": "科幻创作是标准创意写作任务，有成熟流程"
}
```

**2. 加载 Skill**
- 科幻创作 Skill v2.1

**3. 创建 Agent 团队**
- 世界观设计师（设定 2077 北京）
- 大纲规划师（设计故事结构）
- 内容创作者（撰写正文）
- 质量审查员（三层审查）

**4. 执行过程**
```
T+0min:  世界观设计师开始工作
T+15min: 世界观设定完成（2077 北京：飞行器、AI 管家、豆汁儿）
T+15min: 大纲规划师开始工作
T+30min: 大纲完成（单线叙事，1 小时跨度）
T+30min: 内容创作者开始工作
T+60min: 初稿完成（892 字）
T+60min: 质量审查员开始工作
T+70min: 审查通过（92 分），交付
```

### 输出成果

**《清晨的豆汁儿》**（节选）

> 2077 年，北京，清晨五点。
>
> 林默在智能床上自然醒来，床垫检测到她进入浅睡眠阶段，轻柔地释放了唤醒香氛——是淡淡的槐花香，奶奶院子里那棵老槐树的味道。
>
> 窗外，几架个人飞行器无声地滑过天际，在建筑间的空中航道里有序穿行。但在这条胡同里，依然保留着百年前的宁静。远处传来隐约的叫卖声，不是录音，是真实的嗓音——"豆汁儿~~焦圈~~"，是社区里最后一位人工叫卖的王大爷，AI 们默契地为他保留了这片声音的空间。
>
> ...
>
> 科技再变，爱不变。这座城市的清晨，永远有豆汁儿的香气，永远有人在等你回家。

## 📝 示例 2：管理体系设计（创新任务）

### 任务描述

```
为一家 50 人的科技公司设计管理体系
要求：
- 包含组织架构、决策流程、激励机制
- 考虑远程办公和弹性工作制
- 提供实施路线图（3 个月）
- 输出为 Markdown 文档
```

### 执行流程

**1. 任务分类**
```json
{
  "type": "innovative",
  "confidence": 0.95,
  "reasoning": "多领域专业整合，无标准答案，需要创造性设计"

**2. 动态分解**
8 个子任务：
1. 公司现状分析
2. 组织架构设计
3. 决策流程设计
4. 激励机制设计
5. 远程办公政策
6. 弹性工作制设计
7. 实施路线图
8. 风险与调整机制

**3. 创建 Agent 团队**
- 组织管理专家（任务 1、2）
- 流程设计专家（任务 3）
- HR 专家（任务 4）
- 激励设计专家（任务 4）
- 远程协作专家（任务 5、6）
- 变革管理专家（任务 7、8）

**4. 执行计划**
```
阶段 1（串行）：现状分析 → 30 分钟
阶段 2（并行）：架构/决策/激励 → 45 分钟
阶段 3（并行）：远程/弹性 → 30 分钟
阶段 4（串行）：实施路线图 → 30 分钟

## 📝 示例 3：AI 伦理小说（混合任务）

```

- 3 章，每章 1500 字
- 重点描写 AI 伦理冲突
- 采用双主角视角（人类+AI）
- 结局开放式
```

**1. 任务分类**
```json
{
  "type": "hybrid",
  "confidence": 0.92,
  "reasoning": "标准科幻创作 + 定制要求（AI 伦理、双视角、开放式结局）"

```

**2. 加载 Skill + 定制**

**3. 创建 Agent 团队**
- 世界观设计师（标准）
- 大纲规划师（标准）

**第三章：未选择的道路**

## 💡 使用建议

### 选择合适的任务类型

- **简单查询/计算** → 直接描述（自动识别为 simple）
- **创意写作/数据分析** → 明确质量要求（standard）
- **复杂方案设计** → 详细描述需求（innovative）
- **标准 + 定制** → 说明定制部分（hybrid）

### 提供清晰的任务描述

**好的描述：**
```
写一篇科幻短篇小说，主题是 2077 年的北京清晨，要求温暖治愈风格，字数 800-1000 字，需要有明确的世界观设定
```

**不好的描述：**
```
写个小说
```
