# 示例1：科幻小说创作

## 任务描述

```
任务：写一篇800字的科幻短文
主题：2077年的北京
要求：
- 有明确的世界观
- 逻辑严谨
- 有科技感
```

## 系统执行流程

### 步骤1：任务分类

```json
{
  "task-type": "hybrid",
  "confidence": 0.85,
  "matched-skill": "creative-writing",
  "custom-requirements": ["科技感", "世界观设定"],
  "reasoning": "创意写作任务，但有特定主题和世界观要求"
}
```

### 步骤2：任务分解

```json
{
  "subtasks": [
    {
      "id": "subtask-1",
      "name": "世界观设计",
      "agent-role": "世界观设计专家",
      "estimated-time": 60
    },
    {
      "id": "subtask-2",
      "name": "正文撰写",
      "agent-role": "科幻写作专家",
      "input": ["subtask-1.output"],
      "estimated-time": 180
    }
  ],
  "dependency-graph": {
    "edges": [{"from": "subtask-1", "to": "subtask-2"}]
  }
}
```

### 步骤3：子Agent创建

```
创建2个子Agent：
1. 世界观设计专家 (qwen3.5-plus)
2. 科幻写作专家 (qwen3.5-plus)
```

### 步骤4：执行

```
并行组1: [世界观设计专家]
  └─ 输出：世界观设定文档

并行组2: [科幻写作专家]
  ├─ 输入：世界观设定
  └─ 输出：800字科幻短文
```

### 步骤5：质量检查

```
自我检查：写作Agent自检
主Agent确认：检查字数、主题、质量
审查Agent：不启用（子Agent<4）
```

### 步骤6：交付

```
输出：完整的800字科幻短文《2077·北京》
```

### 步骤7：清理

```
删除：世界观设计专家、科幻写作专家
保留：执行日志、Skill计数+1
```

## 实际输出

详见 [2077·北京](./outputs/2077-beijing.md)

## 执行统计

- 总耗时：约5分钟
- 子Agent数：2个
- 修改次数：0次
- 用户满意度：5分

## 经验总结

- 世界观设计环节有效保证了科技感
- 串行执行确保了设定一致性
- 适合类似创意写作任务复用
