# 反思改进器 (Refinement Analyzer)

## 功能

收集任务执行反馈，分析执行数据，识别改进点，优化主Agent的任务理解、角色分工、参数设置等能力。

## 核心职责

| 职责 | 说明 |
|------|------|
| 反馈收集 | 收集用户反馈、子Agent数据、流程数据 |
| 数据分析 | 分析执行效率、质量、问题点 |
| 改进识别 | 识别可改进的环节和方式 |
| 优化建议 | 生成具体的优化建议 |
| 知识沉淀 | 将经验存入经验库 |
| 能力进化 | 更新主Agent的决策模型 |

## 数据分析

### 执行效率分析

```
指标计算：
1. 平均执行时间 = 总时间 / 任务数
2. 阶段耗时占比 = 阶段时间 / 总时间
3. 并行效率 = 实际时间 / 串行预计时间
4. 资源利用率 = 实际使用 / 分配资源

改进识别：
- 某阶段耗时过长 → 优化该阶段
- 并行效率低 → 增加并行度
- 资源利用率低 → 减少资源分配
```

### 质量分析

```
指标计算：
1. 一次性通过率 = 首次通过数 / 总数
2. 平均修改次数 = 总修改数 / 任务数
3. 用户满意度 = 满意度总分 / 任务数
4. 错误率 = 错误数 / 总输出数

改进识别：
- 一次性通过率低 → 加强前期理解
- 平均修改次数多 → 优化检查清单
- 满意度下降 → 分析具体原因
```

### 问题模式识别

```
常见问题模式：
1. 任务理解偏差
   - 症状：多次修改仍不符合要求
   - 原因：需求提取不准确
   
2. 角色分工不合理
   - 症状：某Agent负载过高，其他闲置
   - 原因：任务分解不均衡
   
3. 模型选择不当
   - 症状：输出质量不稳定
   - 原因：模型能力与任务不匹配
   
4. 依赖顺序错误
   - 症状：频繁等待或返工
   - 原因：依赖图设计不合理
```

## 优化策略

### 任务理解优化

**目标：** 提高任务分类和分解的准确性

**方法：**
```
1. 关键词权重调整
   - 分析成功任务的关键词
   - 提高高频关键词的权重
   - 降低低频关键词的权重

2. 分类阈值调整
   - 如误分类多，调整置信度阈值
   - 增加用户确认环节

3. 上下文理解增强
   - 记录用户历史偏好
   - 考虑任务的隐含需求
```

### 角色分工优化

**目标：** 使子Agent分工更合理

**方法：**
```
1. 负载均衡
   - 分析各Agent的工作量
   - 调整任务分配策略

2. 角色合并
   - 如两个角色经常同时空闲
   - 考虑合并为一个Agent

3. 角色拆分
   - 如一个角色负载过高
   - 考虑拆分为两个Agent
```

### 参数设置优化

**目标：** 使超时、并发等参数更合理

**方法：**
```
1. 超时时间调整
   - 统计实际执行时间分布
   - 设置为P90分位数的1.5倍

2. 并发数调整
   - 分析资源利用率
   - 利用率高则增加，低则减少

3. 重试次数调整
   - 分析重试成功率
   - 成功率高则保持，低则减少
```

### 检查清单优化

**目标：** 提高质量检查的准确性

**方法：**
```
1. 检查项增删
   - 新增：多次发现的问题类型
   - 删除：从未触发过的检查项

2. 权重调整
   - 提高关键问题的权重
   - 降低次要问题的权重

3. 阈值调整
   - 根据历史数据调整合格标准
```

## 改进建议生成

### 建议格式

```json
{
  "task-type": "sci-fi-writing",
  "improvements": [
    {
      "category": "task-understanding",
      "priority": "high",
      "description": "增加世界观设定确认环节",
      "reason": "3次任务中有2次世界观设定需要修改",
      "expected-impact": "减少修改次数0.5次/任务",
      "implementation": "在大纲阶段增加世界观检查清单"
    },
    {
      "category": "role-assignment",
      "priority": "medium",
      "description": "拆分写作Agent为初稿和润色两个角色",
      "reason": "写作阶段耗时最长，且修改多",
      "expected-impact": "提升一次性通过率15%",
      "implementation": "任务分解时增加润色环节"
    }
  ]
}
```

### 优先级评估

```
优先级 = 影响程度 × 发生频率 × 实施难度

影响程度：1-5分（5分影响最大）
发生频率：1-5分（5分最频繁）
实施难度：1-5分（1分最容易）

优先级分数 = (影响程度 + 发生频率) × (6 - 实施难度)

高优先级：>30分
中优先级：20-30分
低优先级：<20分
```

## 知识沉淀

### 经验库更新

```json
{
  "experience-type": "success-pattern",
  "task-type": "sci-fi-writing",
  "description": "先确认世界观设定再写大纲，可减少修改",
  "evidence": {
    "task-count": 5,
    "success-rate": 1.0,
    "avg-modifications": 0.6
  },
  "confidence": 0.9,
  "applicable-scenarios": ["科幻小说", "奇幻小说"],
  "timestamp": "2026-04-03T11:10:00"
}
```

### 最佳实践提取

```
从成功任务中提取：
1. 使用的子Agent角色组合
2. 执行顺序
3. 关键检查点
4. 常见陷阱及避免方法

存入最佳实践库，供未来任务参考
```

### 失败案例分析

```
从失败任务中学习：
1. 失败原因分析
2. 可避免的环节
3. 早期预警信号
4. 应对策略

存入失败案例库，避免重复错误
```

## 能力进化

### 决策模型更新

```
更新内容：
1. 任务分类权重
2. 角色分配规则
3. 参数推荐值
4. 检查清单

更新频率：
- 每次任务后：更新计数
- 每10次任务：更新权重
- 每30次任务：更新规则
```

### 个性化适配

```
针对特定用户：
1. 记录用户偏好
2. 分析用户反馈模式
3. 调整输出风格
4. 优化沟通方式

示例：
- 用户A喜欢详细输出 → 增加细节描述
- 用户B喜欢简洁 → 精简输出
- 用户C重视数据 → 增加数据支撑
```

```
验证方法：
1. 对比改进前后的指标
2. A/B测试（如可能）
3. 用户反馈对比

验证周期：
- 短期：下次任务即验证
- 中期：10次任务后验证
- 长期：30次任务后验证
```
