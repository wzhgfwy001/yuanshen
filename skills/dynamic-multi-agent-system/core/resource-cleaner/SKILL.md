# 资源清理器 (Resource Cleaner)

## 功能

任务完成后清理所有临时资源，包括子Agent会话、临时文件、缓存等，确保无资源泄漏。

## 清理范围

### 必须清理的资源

| 资源类型 | 清理方式 | 优先级 |
|----------|----------|--------|
| 临时子Agent | sessions_kill / 自动delete | P0 |
| 审查Agent | 同上 | P0 |
| 临时文件 | 删除文件 | P1 |
| 任务缓存 | 清理缓存 | P1 |
| 会话状态 | 重置状态 | P2 |

### 必须保留的资源

| 资源类型 | 保留原因 |
|----------|----------|
| 已固化的Skill | 长期复用 |
| Skill计数 | 进化追踪 |
| 经验库 | 知识沉淀 |
| 用户确认保留的输出 | 用户需求 |
| 执行日志（最近10次） | 审计和调试 |

---

## 清理流程

### 标准清理流程

```
任务完成确认
    ↓
停止所有子Agent
    ↓
删除子Agent会话
    ↓
清理临时文件
    ↓
清理缓存
    ↓
更新执行日志
    ↓
验证清理结果
    ↓
清理完成
```

### 详细步骤

#### 步骤1：停止子Agent

```json
{
  "action": "stop-all-agents",
  "agent-ids": [
    "agent-search-001",
    "agent-outline-001",
    "agent-writing-001",
    "agent-review-001"
  ],
  "timeout": 30
}
```

#### 步骤2：删除会话

```json
{
  "action": "delete-sessions",
  "session-keys": [
    "session-xxx",
    "session_yyy"
  ],
  "force": true
}
```

#### 步骤3：清理文件

```json
{
  "action": "cleanup-files",
  "paths": [
    "./state/task-001/",
    "./cache/task-001/"
  ],
  "exclude": [
    "*.log",
    "execution-result.json"
  ]
}
```

#### 步骤4：验证清理

```json
{
  "action": "verify-cleanup",
  "check-items": [
    "no-active-agents",
    "no-temp-files",
    "no-cache-left"
  ]
}
```

## 清理规则

### 清理时机

| 时机 | 触发条件 |
|------|----------|
| 任务完成 | 用户确认交付后 |
| 任务失败 | 确认无法继续后 |
| 超时 | 超过最大执行时间 |
| 用户取消 | 用户明确要求取消 |

### 清理延迟

```
清理延迟 = 任务交付后 30秒

原因：
- 给用户时间查看结果
- 允许用户要求修改
- 避免过早清理导致无法回溯
```

### 强制清理

以下情况立即强制清理：

- 系统资源不足（内存<10%）
- 子Agent数量超限（>12个）
- 检测到资源泄漏
- 系统关闭前

## 异常处理

### 清理失败

**现象：** 子Agent删除失败

**处理流程：**
```
1. 记录错误信息
2. 重试1-2次（间隔5秒）
3. 仍失败则：
   - 标记为"僵尸Agent"
   - 记录到待清理队列
   - 下次启动时强制清理
4. 继续清理其他资源
```

### 部分清理

**现象：** 部分资源清理成功，部分失败

**处理流程：**
```
1. 记录成功清理的资源
2. 记录失败清理的资源
3. 生成清理报告
4. 失败资源加入待清理队列
```

### 清理超时

**现象：** 清理操作超过30秒未完成

**处理流程：**
```
1. 终止清理操作
2. 记录超时资源
3. 加入待清理队列
4. 下次启动时处理
```

## 待清理队列

### 队列结构

```json
{
  "pending-cleanup": [
    {
      "resource-type": "agent",
      "resource-id": "agent-xxx",
      "task-id": "task-001",
      "failed-at": "2026-04-03T11:00:00",
      "retry-count": 2,
      "last-error": "session not found",
      "next-retry": "2026-04-03T12:00:00"
    }
  ]
}
```

### 队列处理

- 处理时机：系统启动时、定时任务（每小时）
- 最大重试次数：5次
- 超过5次：记录日志，标记为"无法清理"

## 资源监控

### 监控指标

| 指标 | 阈值 | 告警 |
|------|------|------|
| 活跃子Agent数 | >12 | 警告 |
| 临时文件大小 | >100MB | 警告 |
| 待清理队列长度 | >10 | 警告 |
| 清理失败率 | >10% | 警告 |

### 监控频率

- 实时监控：活跃子Agent数
- 每分钟：临时文件大小
- 每小时：待清理队列
- 每天：清理失败率统计

## 僵尸Agent处理

### 识别标准

符合以下任一条件：

- 会话存在但无响应
- 状态为"running"但超过1小时无输出
- 清理时返回"not found"但会话列表中存在

### 处理流程

```
1. 标记为僵尸Agent
2. 尝试强制终止（kill -9）
3. 从会话列表中移除
4. 记录到僵尸Agent日志
5. 下次启动时再次检查
```

## 最佳实践

### 预防资源泄漏

1. **创建时指定cleanup**：sessions_spawn时设置`cleanup: "delete"`
2. **设置超时**：runTimeoutSeconds避免无限等待
3. **定期检查**：每小时检查活跃Agent数
4. **及时清理**：任务完成后立即清理

### 清理优化

1. **批量清理**：多个任务完成后批量清理
2. **异步清理**：不阻塞主流程
3. **优先级清理**：先清理大资源
4. **重试退避**：失败后指数退避重试
