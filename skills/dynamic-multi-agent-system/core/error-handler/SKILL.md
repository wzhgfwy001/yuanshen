## 处理分类

| 分类 | 说明 | 适用场景 |
|------|------|----------|
| RETRY | 重试 | 网络波动、临时故障 |
| SKIP | 跳过 | 可选操作、非关键错误 |
| ABORT | 中止 | 严重错误、权限问题 |

## 最佳实践

### ✅ 推荐
1. **使用统一错误码** - ERR-{Category}-{Code} 格式
2. **记录完整上下文** - 错误发生时记录足够调试信息
3. **用户友好提示** - 对外显示友好错误消息
4. **分级处理** - RETRY/SKIP/ABORT 严格执行
5. **监控错误率** - 持续跟踪错误统计

### ❌ 避免
1. **吞掉错误** - 不要忽略或隐藏错误
2. **无限重试** - 设置重试上限
3. **直接暴露内部错误** - 用户不应看到技术细节
4. **重复记录日志** - 同一错误不要多次记录
