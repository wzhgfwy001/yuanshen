# 💻 代码审查检查清单

**版本：** v1.0  
**适用场景：** 代码审查/代码质量检查  
**检查阶段：** 提交前/合并前

---

## 🔍 代码质量检查

### 代码规范
- [ ] 是否遵循编码规范？
- [ ] 命名是否有意义？
- [ ] 缩进是否一致？
- [ ] 注释是否充分？
- [ ] 代码格式是否统一？

### 代码结构
- [ ] 函数是否过长（<50 行）？
- [ ] 类是否过大（<500 行）？
- [ ] 模块是否单一职责？
- [ ] 依赖是否合理？
- [ ] 是否有重复代码？

### 可读性
- [ ] 代码是否易读？
- [ ] 逻辑是否清晰？
- [ ] 变量名是否达意？
- [ ] 注释是否准确？
- [ ] 是否有魔法数字？

---

## 🛡️ 安全性检查

### 输入验证
- [ ] 是否验证用户输入？
- [ ] 是否检查边界条件？
- [ ] 是否处理空值？
- [ ] 是否处理异常格式？
- [ ] 是否限制输入长度？

### 数据安全
- [ ] 敏感数据是否加密？
- [ ] 密码是否哈希存储？
- [ ] API 密钥是否保护？
- [ ] SQL 是否参数化？
- [ ] XSS 是否防护？

### 权限控制
- [ ] 是否验证用户权限？
- [ ] 是否检查资源访问？
- [ ] 是否有越权风险？
- [ ] 日志是否记录操作？
- [ ] 敏感操作是否二次确认？

---

## ⚡ 性能检查

### 算法效率
- [ ] 算法复杂度是否合理？
- [ ] 是否有不必要的循环？
- [ ] 是否有重复计算？
- [ ] 是否使用合适数据结构？
- [ ] 是否有性能瓶颈？

### 资源使用
- [ ] 内存是否泄漏？
- [ ] 连接是否关闭？
- [ ] 文件句柄是否释放？
- [ ] 缓存是否合理？
- [ ] 是否有资源浪费？

### 数据库优化
- [ ] 查询是否优化？
- [ ] 索引是否使用？
- [ ] N+1 查询是否避免？
- [ ] 事务是否合理？
- [ ] 锁是否正确使用？

---

## 🧪 测试检查

### 单元测试
- [ ] 是否有单元测试？
- [ ] 覆盖率是否达标（>80%）？
- [ ] 边界条件是否测试？
- [ ] 异常情况是否测试？
- [ ] 测试是否独立？

### 集成测试
- [ ] 接口是否测试？
- [ ] 模块交互是否测试？
- [ ] 端到端是否测试？
- [ ] 性能是否测试？
- [ ] 压力是否测试？

### 测试质量
- [ ] 测试是否可重复？
- [ ] 测试是否稳定？
- [ ] 断言是否准确？
- [ ] 测试数据是否隔离？
- [ ] 测试是否快速？

---

## 🐛 错误处理检查

### 异常处理
- [ ] 是否捕获异常？
- [ ] 异常是否记录？
- [ ] 是否给用户友好提示？
- [ ] 是否恢复或降级？
- [ ] 是否避免吞掉异常？

### 日志记录
- [ ] 日志级别是否合适？
- [ ] 是否记录关键信息？
- [ ] 是否避免敏感信息？
- [ ] 日志格式是否统一？
- [ ] 日志是否可查询？

### 错误恢复
- [ ] 是否有重试机制？
- [ ] 是否有熔断机制？
- [ ] 是否有降级方案？
- [ ] 是否有监控告警？
- [ ] 是否有回滚方案？
