# 任务分类器 (Task Classifier)

## 功能

判断用户输入的任务类型，选择相应的处理模式。

## 四种任务类型

### 类型1：单一任务 (Simple Task)

**判断标准：**
- 任务单维度（只涉及一个领域）
- 不需要多角色协作
- 主Agent可在合理时间内完成（<5分钟）
- 输出格式简单（文本、计算结果等）

**关键词特征：**
- "翻译"、"总结"、"计算"、"解释"、"定义"
- 无复杂约束条件
- 无多步骤要求

**示例：**
```
✅ "把这段文字翻译成英文"
✅ "帮我总结这篇文章"
✅ "计算这组数据的平均值"
✅ "解释什么是量子力学"
```

**处理方式：** 主Agent直接执行或创建1个子Agent

**置信度评分：** 关键词匹配+约束分析

---

### 类型2：标准任务 (Standard Task)

**判断标准：**
- 任务类型在Skill库中有匹配
- 执行次数≥3次（已验证的Skill）
- 用户没有特殊要求修改流程
- 有成熟的执行模板

**关键词特征：**
- 常见任务类型（写小说、分析数据、写代码）
- 无特殊定制要求
- 符合常规预期

**示例：**
```
✅ "写一个科幻小说"（已有科幻创作Skill）
✅ "分析销售数据并生成报告"（已有数据分析Skill）
✅ "开发一个待办事项应用"（已有代码开发Skill）
```

**处理方式：** 加载Skill，按固化流程执行

**Skill匹配逻辑：**
1. 提取任务关键词
2. 在Skill库中查找匹配
3. 检查执行次数≥3
4. 确认无特殊定制要求

---

### 类型3：创新任务 + 标准任务 (Hybrid Task)

**判断标准：**
- 主体任务有对应Skill
- 但需要特殊处理或定制
- 在标准流程基础上增加定制步骤
- 差异度20-50%

**关键词特征：**
- "重点描写..."、"特别关注..."、"需要..."
- 额外的约束条件
- 非标准输出要求

**示例：**
```
✅ "写一本科幻小说，重点描写科技细节"
✅ "分析数据，但要对比行业基准"
✅ "开发一个应用，要求支持离线模式"
```

**处理方式：** 加载Skill，添加定制环节

**定制识别：**
1. 识别标准部分
2. 识别定制要求
3. 评估定制复杂度
4. 决定是否需要额外子Agent

---

### 类型4：创新任务 (Innovative Task)

**判断标准：**
- 任务类型在Skill库中无匹配
- 或者虽然类型匹配但有重大变化（差异>50%）
- 需要全新流程或角色分工
- 多步骤、多维度、复杂约束

**关键词特征：**
- 新颖的任务描述
- 多个独立要求
- 复杂约束条件
- 无先例可循

**示例：**
```
✅ "帮我设计一套完整的公司管理体系"
✅ "写一个推理小说，采用叙事诡计+时间诡计"
✅ "策划一场线上线下结合的营销活动"
```

**处理方式：** 动态组建，创建子Agent团队

**分解策略：**
1. 识别任务维度
2. 分析依赖关系
3. 确定子Agent角色
4. 规划执行顺序

## 分类算法

### 步骤1：关键词提取

```
输入：任务描述文本
输出：关键词列表

方法：
1. 分词
2. 去除停用词
3. 提取名词、动词
4. 识别任务类型关键词
```

### 步骤2：Skill匹配

```
输入：关键词列表
输出：匹配的Skill（如有）

方法：
1. 遍历Skill库
2. 计算关键词相似度
3. 返回最佳匹配（相似度>70%）
```

### 步骤3：定制分析

```
输入：任务描述 + 匹配Skill
输出：定制要求列表

方法：
1. 识别"重点"、"特别"、"需要"等关键词
2. 提取后续内容
3. 评估定制复杂度
```

### 步骤4：置信度计算

```
置信度 = 基础匹配度 × 定制调整系数

基础匹配度：
- 关键词匹配：40%
- Skill匹配：40%
- 历史执行：20%

定制调整系数：
- 无定制：1.0
- 轻微定制：0.9
- 中等定制：0.7
- 重大定制：0.5
```

### 步骤5：类型判定

```
if 置信度 >= 0.8 and 无定制:
    return 类型1或类型2
elif 置信度 >= 0.7 and 轻微定制:
    return 类型3
elif 置信度 < 0.7 or 重大定制:
    return 类型4
else:
    return 类型3（保守策略）
```

## 输出格式

```json
{
  "task-type": "simple|standard|hybrid|innovative",
  "confidence": 0.85,
  "matched-skill": "sci-fi-writing",
  "custom-requirements": ["重点描写科技细节"],
  "recommended-agents": ["写作专家", "技术顾问"],
  "estimated-complexity": "medium",
  "reasoning": "关键词匹配科幻创作，但有技术细节定制要求"
}
```

---

## 置信度阈值

| 阈值 | 含义 | 处理 |
|------|------|------|
| >= 0.8 | 高置信度 | 直接分类 |
| 0.7-0.8 | 中置信度 | 分类，标记可能需要确认 |
| < 0.7 | 低置信度 | 向用户确认任务类型 |

---

## 异常处理

### 无法分类

**现象：** 置信度<0.5，无匹配Skill

**处理：**
1. 向用户确认："这是一个新类型的任务，请问您希望如何处理？"
2. 提供选项：
   - A) 按创新任务处理（动态组建）
   - B) 按标准任务处理（使用最接近的Skill）
   - C) 手动指定处理方式

### 多类型模糊

**现象：** 同时匹配多个Skill，置信度接近

**处理：**
1. 列出匹配的Skill
2. 向用户确认："检测到多个可能的处理方式，请选择："
3. 根据用户选择确定类型
