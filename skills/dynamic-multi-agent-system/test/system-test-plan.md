## 测试用例

### 用例 1：单一任务测试

**任务：** 翻译一段中文为英文

**输入：**
```
任务：翻译以下文字为英文
内容："2077 年的北京清晨，科技与传统和谐共存。"
要求：准确传达原意，保持文学性
```

**预期流程：**
1. 分类器识别为 `simple` 类型
2. 主 Agent 直接执行或创建 1 个子 Agent
3. 输出翻译结果
4. 质量检查（自检 + 主 Agent 确认）

**验证要点：**
- ✅ 任务类型识别正确（simple）
- ✅ 无需多 Agent 协作
- ✅ 翻译质量准确
- ✅ 执行时间短（<5 分钟）

---

### 用例 2：标准任务测试（已有）

**任务：** 创作科幻短篇小说

**输入：**
```
任务：写一篇科幻短篇小说
主题：2077 年的北京清晨
要求：800-1000 字，温暖治愈风格
```

**预期流程：**
1. 分类器识别为 `standard` 类型
2. 加载已有 Skill（如有）或按标准流程
3. 4 个子 Agent 协作（世界观→大纲→写作→审查）
4. 输出完整小说

**验证要点：**
- ✅ 任务类型识别正确（standard）
- ✅ 使用标准流程
- ✅ 输出质量稳定（≥85 分）
- ✅ 执行时间合理（~70 分钟）

**状态：** ✅ 已完成（2026-04-04，92 分）

### 用例 3：创新任务测试

**任务：** 设计一套完整的公司管理体系

**输入：**
```
任务：为一家 50 人的科技公司设计管理体系
要求：
- 包含组织架构、决策流程、激励机制
- 考虑远程办公和弹性工作制
- 提供实施路线图（3 个月）
- 输出为 Markdown 文档
```

**预期流程：**
1. 分类器识别为 `innovative` 类型（置信度≥0.8）
2. 任务分解器动态分解（6-8 个子任务）
3. 创建 5-6 个子 Agent（组织专家、HR 专家、流程专家等）
4. 执行协调器调度（并行 + 串行混合）
5. 质量检查器三层审查
6. 输出完整管理体系文档

**验证要点：**
- ✅ 任务类型识别正确（innovative）
- ✅ 动态分解合理（6-8 个子任务）
- ✅ 子 Agent 角色专业（5-6 个）
- ✅ 依赖关系正确
- ✅ 输出完整（覆盖所有要求）
- ✅ 质量达标（≥85 分）

### 用例 4：混合任务测试

**任务：** 写一本科幻小说，重点描写人工智能伦理

**输入：**
```
任务：创作中篇科幻小说
主题：人工智能伦理困境
要求：
- 3 章，每章 1500 字
- 重点描写 AI 伦理冲突
- 采用双主角视角（人类+AI）
- 结局开放式
```

**预期流程：**
1. 分类器识别为 `hybrid` 类型（标准 + 定制）
2. 加载科幻创作 Skill
3. 添加定制环节（AI 伦理顾问）
4. 5 个子 Agent 协作
5. 输出小说

**验证要点：**
- ✅ 任务类型识别正确（hybrid）
- ✅ 标准流程 + 定制环节
- ✅ AI 伦理描写深入
- ✅ 双主角视角清晰
