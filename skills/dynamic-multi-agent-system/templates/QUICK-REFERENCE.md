## 一句话启动（最常用）

| 需求 | 模板 |
|------|------|
| 写文章 | "写一篇[主题]文章，[字数]字" |
| 写代码 | "用[语言]开发[功能]" |
| 修bug | "帮我修复bug：[错误]" |
| 做分析 | "分析[对象]的[维度]" |
| 写小说 | "写一个[类型]小说，[字数]字" |
| 翻译 | "翻译[语言]到[语言]：[内容]" |
| 做计划 | "制定[目标]的计划" |
| 解释概念 | "用简单的话解释[概念]" |

---

## 任务复杂度速判

```
一句话能说清楚 → 单一任务 ⭐
需要几步处理 → 标准任务 ⭐⭐
需要创意/分析 → 创新任务 ⭐⭐⭐
多角色协作/长篇 → 复杂任务 ⭐⭐⭐⭐
```

---

## 常用约束词

| 约束 | 用法 |
|------|------|
| 字数 | "500字左右" / "至少2000字" |
| 风格 | "口语化" / "专业" / "轻松" |
| 受众 | "给投资人看" / "面向00后" |
| 格式 | "输出JSON" / "Markdown格式" |
| 禁止 | "不要用AI味" / "别用专业术语" |

## 常用输出格式

| 格式 | 用法 |
|------|------|
| Markdown | 文档、报告、文章 |
| JSON | 数据、配置文件 |
| 代码块 | 代码、脚本 |
| 表格 | 对比、数据 |
| 分点列表 | 要点、建议 |

---

## 提示技巧

✅ **越具体越好** — 主题明确胜过泛泛而谈
✅ **说明用途** — "给甲方汇报" vs "随便写写"
✅ **指定格式** — 省去后续调整的时间
✅ **设定限制** — "不用AI相关词汇"

❌ **太模糊** — "帮我写点东西"
❌ **太复杂** — 先拆解再启动
