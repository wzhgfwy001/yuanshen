## 反馈收集流程

```
用户反馈
   ↓
ClawHub 评论/反馈模板
   ↓
保存到 feedbacks/YYYY-MM/ 目录
   ↓
分类记录到 USER-FEEDBACK.md
   ↓
24-72h 内回复
   ↓
每周汇总到周报
```

---

## 优先级定义

- **P0 紧急：** 系统崩溃、数据丢失、安全漏洞（24h 内响应）
- **P1 高：** 功能异常、性能问题（48h 内响应）
- **P2 中：** 使用疑问、配置问题（72h 内响应）
- **P3 低：** 功能建议、优化建议（下周迭代）

---

## 反馈处理流程

1. **收集** - 从 SkillHub 评论/私信收集反馈
2. **分类** - 按类型和优先级分类
3. **响应** - 在规定时间内回复用户
4. **处理** - 开发团队处理问题
5. **验证** - 验证修复效果
6. **关闭** - 通知用户并关闭工单
