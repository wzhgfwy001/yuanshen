## 🎯 反馈处理 SOP

### 步骤 1：接收（实时）

**微信：**
- 及时通过好友请求
- 接收反馈信息
- 保存聊天记录（重要）

**邮箱：**
- 每日 09:00/18:00 检查
- 阅读邮件内容
- 标记未读→已读

---

### 步骤 2：记录（接收后立即）

**操作：**
1. 打开 `operations/USER-FEEDBACK.md`
2. 添加新记录到表格
3. 分配 ID（按顺序）
4. 标记状态为"待处理"

**记录内容：**
- 日期时间
- 反馈渠道（微信/邮箱）
- 用户标识（昵称/邮箱）
- 反馈类型
- 优先级（P0-P3）
- 内容摘要

---

### 步骤 3：分类定级（记录后立即）

**优先级标准：**

| 级别 | 类型 | 响应时间 | 示例 |
|------|------|----------|------|
| **P0** | 紧急 Bug | 24h | 系统崩溃、数据丢失 |
| **P1** | 高优先级 | 48h | 功能异常、严重 Bug |
| **P2** | 中优先级 | 72h | 使用问题、一般建议 |
| **P3** | 低优先级 | 下周迭代 | 优化建议、小功能 |

---

### 步骤 4：响应（按优先级）

**P0（24h 内）：**
```
您好，感谢反馈！

您遇到的问题是 [问题描述]，这是我们的责任。
我们已确认为 Bug，正在紧急修复。
预计 [时间] 内修复完成，修复后会第一时间通知您。

如有任何问题，随时联系我。
再次感谢您的反馈！
```

**P1（48h 内）：**
```
您好，感谢反馈！

您提到的 [问题/建议] 已收到。
我们正在分析/评估，预计 [时间] 内给您回复。

如有补充信息，欢迎随时提供。
感谢支持！
```

**P2（72h 内）：**
```
您好，感谢反馈！

关于 [问题/建议]：
- 如是问题：解答 + 文档链接
- 如是建议：已记录，会在下次迭代评估

感谢支持！
```

**P3（周报中感谢）：**
```
在周报/更新日志中提及：
- 感谢用户 XX 的建议
- 已纳入需求池
```

---

### 步骤 5：处理（按类型）

**Bug 报告：**
```
1. 复现问题
2. 定位原因
3. 修复代码
4. 测试验证
5. 通知用户
6. 更新版本（如需）
```

**功能建议：**
```
1. 评估可行性
2. 评估工作量
3. 排期（本周/下周/未来）
4. 实现功能
5. 测试验证
6. 通知用户
```

**使用问题：**
```
1. 详细解答
2. 提供文档/示例
3. 确认用户理解
4. 更新 FAQ（如常见问题）
```

**性能反馈：**
```
1. 收集使用数据
2. 分析性能瓶颈
3. 优化代码
4. 测试对比
5. 通知用户
```

---

### 步骤 6：关闭（用户确认后）

**条件：**
- 问题已解决
- 用户确认满意
- 功能已实现

**操作：**
1. 更新状态为"已关闭"
2. 记录关闭时间
3. 添加到贡献者名单
4. 周报中感谢
