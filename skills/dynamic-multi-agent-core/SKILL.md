
# 混合动态多Agent运行系统 - 核心Skill

## 概述

这是一个通用的元级别系统，让主Agent能够动态创建、管理多个子Agent协作完成任务。

## 核心原则

### 1. 任务分类（四种类型）

| 类型 | 定义 | 处理方式 |
|------|------|----------|
| 单一任务 | 单维度，无需多Agent | 主Agent直接处理 |
| 标准任务 | 已有成熟Skill（≥3次验证） | 加载Skill执行 |
| 创新+标准 | 主体有Skill，需定制 | 加载+微调 |
| 创新任务 | 全新技术/场景 | 动态组建子Agent团队 |

### 2. 子Agent数量（动态分配）

基于任务上下文分解，按需分配：
- **简单任务**：1-2个Agent
- **中等任务**：3个Agent
- **复杂任务**：4-6个Agent

### 3. 审查机制（三层检查）




子Agent执行 → ①自我检查 → ②主Agent确认 → ③审查Agent审查

### 4. 资源清理（强制）
- 任务完成后**必须删除**所有临时子Agent
- 审查Agent经验教训存入主Agent记忆
## 关键API
### 创建子Agent
使用 `sessions_spawn` 创建临时子Agent：


javascript
{
task: "任务描述",
model: "modelstudio/qwen3.5-plus",  // 可指定不同模型
cleanup: "delete",  // 任务完成后自动删除
label: "agent-角色名称"
}

### 管理子Agent
- `sessions_list` - 查看活跃子Agent
- `subagents kill` - 删除指定子Agent
- `sessions_send` - 向子Agent发送消息
## 执行流程
### 第一步：任务分析
1. 理解用户需求
2. 判断任务类型（单一/标准/创新+标准/创新）
3. 如需多子Agent，进行任务分解
### 第二步：组建团队
根据任务复杂度，动态创建子Agent：
- 明确每个Agent的角色和职责
- 分配适当的模型（语言能力/代码能力/创意能力）
- 设置清晰的输入输出期望
### 第三步：执行与检查
1. 子Agent执行任务 → 自我检查
2. 主Agent审查输出 → 确认/要求修改
3. 复杂任务设立审查Agent → 全面审查
### 第四步：整合交付
1. 整合所有子Agent输出
2. 最终质量检查
3. 交付用户
### 第五步：清理资源
1. 删除所有临时子Agent
2. 提取经验教训存入记忆
3. 更新Skill（如已满足固化条件）
## Skill固化规则
- **触发条件**：成功执行3次主任务
- **固化内容**：模板流程 + 审查功能（功能，非Agent）
- **存储位置**：skills/library/
## 注意事项
1. **不要固定子Agent数量** - 根据任务按需分配
2. **每环节都要检查** - 而非只在最后检查
3. **必须清理资源** - 避免资源泄漏
4. **记录经验教训** - 持续优化系统
