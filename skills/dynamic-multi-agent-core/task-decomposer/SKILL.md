# 任务分解器 - 子Skill

## 用途
将复杂任务分解为可执行的子任务，确定需要多少子Agent以及每个Agent的角色。

## 输入
- 用户原始需求
- 任务类型判断结果

## 输出
- 子任务列表
- 每个子任务的Agent角色建议
- 子任务执行顺序（并行/串行）

## 分解原则
### 1. 按能力匹配角色
| 任务性质 | 推荐Agent类型 | 适用模型 |
|----------|---------------|----------|
| 信息搜索/调研 | 搜索Agent | 搜索能力强 |
| 结构化大纲 | 规划Agent | 逻辑清晰 |
| 内容创作 | 写作Agent | 创意丰富 |
| 代码开发 | 编码Agent | 代码能力 |
| 质量审查 | 审查Agent | 客观严格 |

### 2. 确定执行顺序
- **并行执行**：子任务之间相互独立
- **串行执行**：子任务有依赖关系（上游输出作为下游输入）
