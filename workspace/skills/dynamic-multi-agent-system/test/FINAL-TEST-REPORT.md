## 发现并修复的问题

### 问题1：selectPrimaryLevel函数逻辑错误 [高优先级]
**位置：** `core/diagnosis/diagnose.js`

**问题描述：**
```javascript
// 原代码
let primary = 'L0';
let highestScore = scores['L0'] || 0;  // 当L0不在scores中时，highestScore=0
```
当L0不在scores中时，`highestScore = 0`，导致后续比较无法正确选择其他层级。

**修复方案：**
```javascript
// 修复后：找出得分最高的有效层级
let primary = null;
let bestScore = 0;
for (const [level, score] of Object.entries(scores)) {
    if (score < 0.6) continue;  // 过滤信号不足的层级
    if (primary === null || score > bestScore) {
        primary = level;
        bestScore = score;
    }
}
return primary || 'L0';  // 无有效层级时返回L0
```

**影响：** 修复前所有输入都错误地返回L0，修复后正确识别各层级问题。

---

### 问题2：missing检测逻辑错误 [高优先级]

**位置：** `core/diagnosis/diagnose.js` - `detectSignals`函数

**问题描述：**
使用英文单词（'goal', 'purpose'等）去检测中文文本，导致所有中文输入都错误触发L1的"缺少goal/purpose"信号。

**修复方案：**
1. 删除missing检测作为独立信号源
2. 只保留关键词和模式匹配作为有效信号
3. 关键词检测已有明确信号时，missing作为辅助参考

**影响：** 修复前所有中文任务都错误识别为L1需求不清。

---

### 问题3：L2关键词过于宽泛 [中优先级]

**位置：** `core/diagnosis/diagnose.js` - `DETECTION_SIGNALS.L2`

**问题描述：**
L2关键词包含"包括"、"什么的"，这些词在正常功能列举时也会触发，导致误判。

**修复方案：**
```javascript
// 修复前
keywords: ['等等', '还有', '以及', '包括', '什么的', '等等看']

// 修复后：只保留明确的范围扩展词
keywords: ['等等', '还有', '以及', '还要', '再加', '另外']
```

**影响：** 修复前"做个网站，包括登录注册"被误判为L2范围模糊。

---

### 问题4：L3关键词与L4冲突 [中优先级]

**位置：** `core/diagnosis/diagnose.js` - `DETECTION_SIGNALS.L3`

**问题描述：**
L3关键词"帮我"、"做个"过于通用，与L4（产品类型）判断冲突。

**修复方案：**
```javascript
// 修复前
keywords: ['帮我', '做个', '设计下', '写个', '搞个', '弄个']

// 修复后：更具体的实施相关词汇
keywords: ['程序', '代码', '算法', '数据库', '接口', '系统', '分析', '处理', '开发', '设计']
```

**影响：** 修复前"做个App"错误识别为L3实施模糊，正确应为L4产品空泛。

---

### 问题5：L3缺少关键实施词汇 [低优先级]

**位置：** `core/diagnosis/diagnose.js` - `DETECTION_SIGNALS.L3`

**问题描述：**
L3关键词缺少"分析"等常见实施相关词汇。

**修复方案：**
在keywords中增加：'分析', '处理', '开发', '设计'

**影响：** 修复前"帮我分析销售数据"被错误识别为L0，正确应为L3。
