# API 参考

## 主接口

### 任务提交

**入口：** 直接向主 Agent 发送任务消息

```
任务：[任务描述]
要求：
- [要求 1]
- [要求 2]
```

**示例：**
```
任务：写一本科幻小说，主题"2077 年的北京"
要求：
- 10 章，每章 1000 字
- 硬科幻风格
- 包含科技细节描写
```

## 配置接口

### 系统配置

位置：`~/.openclaw/openclaw.json`

```json
{
  "multi-agent": {
    "max-concurrent-tasks": 3,
    "max-sub-agents": 12,
    "enable-evolution": true,
    "confidence-threshold": 0.7,
    "enable-reviewer": true,
    "reviewer-threshold": 4
  }
}
```

### 配置项说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| max-concurrent-tasks | number | 3 | 最多并发主任务数 |
| max-sub-agents | number | 12 | 最多子 Agent 总数 |
| enable-evolution | boolean | true | 启用 Skill 进化 |
| confidence-threshold | number | 0.7 | 任务分类置信度阈值 |
| enable-reviewer | boolean | true | 启用审查 Agent |
| reviewer-threshold | number | 4 | 启用审查的子 Agent 数量阈值 |

## 控制接口

### 暂停任务

**命令：**
```
暂停当前任务
```

**效果：**
- 停止创建新的子 Agent
- 等待进行中的子 Agent 完成
- 保存当前状态

### 恢复任务

**命令：**
```
继续执行
```

**效果：**
- 从暂停点继续
- 恢复子 Agent 创建

### 取消任务

**命令：**
```
取消当前任务
```

**效果：**
- 终止所有进行中的子 Agent
- 清理临时资源
- 保存部分结果（如有）

## 事件监听

### 进度事件

系统会定期报告进度：

```
[进度] 任务 1/4 完成：科幻设定专家
[进度] 任务 2/4 完成：大纲设计专家
[进度] 任务 3/4 进行中：小说写作专家 (预计 2 分钟)
```

### 完成事件

```
✅ 任务完成！
- 总耗时：35 分钟
- 创建子 Agent：4 个
- 输出：10000 字
- 质量检查：通过
```

### 错误事件

```
❌ 任务失败
- 失败阶段：小说写作专家
- 错误：超时（超过 5 分钟）
- 建议：重试或简化要求
```

## 高级用法

### 自定义 Agent 角色

**命令：**
```
任务：[任务描述]
创建以下专家：
1. [角色名 1] - 负责 [职责 1]
2. [角色名 2] - 负责 [职责 2]
```

**示例：**
```
任务：设计游戏世界观
创建以下专家：
1. 历史学家 - 设计世界历史脉络
2. 地理学家 - 设计大陆和气候
3. 社会学家 - 设计种族和文化
4. 经济学家 - 设计贸易和资源
```

### 指定执行顺序

**命令：**
```
任务：[任务描述]
执行顺序：
1. [步骤 1] → 2. [步骤 2] → 3. [步骤 3]
```

### 并行执行要求

**命令：**
```
任务：[任务描述]
以下部分并行执行：
- [部分 A]
- [部分 B]
- [部分 C]
最后合并结果
```

## 调试接口

### 启用详细日志

**命令：**
```
设置：启用详细日志
```

**效果：**
- 记录每个子 Agent 的完整 prompt 和输出
- 保存到 `state/execution-logs/`

### 查看调试信息

**命令：**
```
调试：显示当前任务状态
```

**返回：**
```json
{
  "task-id": "task-005",
  "phase": "子 Agent 执行",
  "completed": 2,
  "total": 4,
  "current-agent": "小说写作专家",
  "elapsed": "15 分钟"
}
```
