### 第 2 步：使用最佳实践

#### ✅ 推荐做法

**1. 相似任务连续做**
```
好：连续翻译 10 句话 → 批量处理，节省 60%
不好：翻译 1 句，做别的，再翻译 1 句 → 无法批量
```

**2. 长任务分阶段**
```
好：先大纲→确认→填充 → 避免返工，节省 50%
不好：一次性生成→不满意→重写 → 浪费 Token
```

**3. 复用历史结果**
```
好：相似任务自动复用 → 节省 50-60%
不好：每次都重新生成 → 浪费 Token
```

**4. 设置合理预算**
```
好：设置预算上限 → 防止超支
不好：无限制 → 可能浪费
```

---

#### ❌ 避免做法

**1. 频繁切换任务类型**
```
不好：翻译→写代码→翻译→写文章 → 无法批量
好：集中完成同类任务 → 批量处理
```

**2. 需求不明确**
```
不好："写点什么" → 可能返工
好："写 800 字科幻短文，2077 年北京" → 一次通过
```

**3. 过度要求完美**
```
不好：每句都修改 → 反复调用
好：先完成再优化 → 渐进式处理
```

### 技巧 2：任务聚类

```yaml
# 将相似任务分组处理
clustering:
  by_type:        # 按类型分组
    - 写作类
    - 代码类
    - 分析类
  
  by_complexity:  # 按复杂度分组
    - 简单（<4 分）
    - 中等（4-7 分）
    - 复杂（>7 分）
  
  by_urgency:     # 按紧急程度分组
    - 紧急（今天）
    - 普通（本周）
    - 不紧急（本月）
```

**效果：** 批量处理率提升至 80%

### 技巧 3：智能调度

```yaml

  peak_hours:       # 高峰期用轻量模型

    max_batch_size: 20
```

## ⚠️ 注意事项

### 1. 缓存管理

```yaml
# 定期清理过期缓存
cache_cleanup:
  max_age: 30d        # 超过 30 天清理
  max_size: 10000     # 最多 10000 条
  on_full: lru        # 满时删除最少使用
```

**建议：** 每月检查一次缓存使用
