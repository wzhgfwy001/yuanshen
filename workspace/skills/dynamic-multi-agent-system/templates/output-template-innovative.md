# 创新任务输出模板

## 适用场景

- 类型4：创新任务 (Innovative Task)
- 诊断结果：L0/L5 - 执行明确或UX不全
- 复杂度：高

## 输出结构

```
## 🔍 任务诊断
[瓶颈诊断结果摘要]

## 📊 任务分类
[分类结果和处理策略]

## 📋 执行摘要
[整体工作概述]

## 🎯 关键决策
[主要设计/架构决策]

## 📌 核心产出
[最重要的3个产出]

## 📄 详细内容
[完整内容]

## 🔄 执行过程
[关键步骤回顾]

## 💡 后续建议
[可选：后续优化方向]
```

## 各部分说明

### 任务诊断（创新任务独有）

```
格式：卡片式
- 瓶颈层级：L?
- 置信度：??
- 主要挑战：?
```

### 任务分类（创新任务独有）

```json
{
  "task-type": "innovative",
  "complexity": "high",
  "agents-used": ["agent1", "agent2"],
  "strategy": "dynamic-team"
}
```

### 执行摘要

```
格式：多行概述
- 完成的工作
- 参与的角色
- 关键产出
```

### 关键决策

```
格式：决策 + 理由
- 每个决策一行
- 说明选择原因
```

### 核心产出

```
格式：优先级列表
1. [最重要产出]
2. [次重要产出]
3. [第三重要产出]
```

### 详细内容

```
格式：完整展开
- 章节式结构
- 代码块标注
- 图表说明
```

### 执行过程

```
格式：时间线/步骤流
- 主要阶段
- 关键节点
- 审查点
```

### 后续建议

```
格式：建议 + 理由
- 可选的后续优化
- 预期的改进效果
```

## 示例

```
## 🔍 任务诊断

| 项目 | 值 |
|------|-----|
| 瓶颈层级 | L4 - 产品空泛 |
| 置信度 | 78% |
| 主要挑战 | 产品形态不具体，用户场景缺失 |

## 📊 任务分类

```json
{
  "task-type": "innovative",
  "complexity": "high",
  "agents-used": ["产品经理", "UX设计师", "技术架构师"],
  "strategy": "动态组建团队"
}
```

## 📋 执行摘要

完成了"智能备课助手"产品设计，包括：
- 3个子Agent协作：产品经理(需求分析) + UX设计师(交互设计) + 技术架构师(技术方案)
- 完整的PRD文档、UX原型描述、技术架构图
- 总计5个核心功能模块设计

## 🎯 关键决策

1. **采用对话式交互** - 教师习惯自然语言，对话降低学习成本
2. **本地部署优先** - 教育数据敏感，云端有合规风险
3. **模块化架构** - 支持按需启用功能，适应不同学校需求

## 📌 核心产出

1. 📄 PRD文档（产品需求定义）
2. 🎨 UX设计方案（交互流程+页面结构）
3. 🏗️ 技术架构图（系统组件+数据流）

## 📄 详细内容

### 产品需求文档（PRD）

[完整PRD内容...]

### UX设计方案

[UX设计详情...]

### 技术架构

[技术架构详情...]

## 🔄 执行过程

1. **需求分析** (产品经理Agent)
   - 用户调研整合
   - 核心场景提取
   
2. **产品设计** (产品经理Agent)
   - 功能列表
   - PRD撰写

3. **UX设计** (UX设计师Agent)
   - 交互流程
   - 页面结构

4. **技术架构** (技术架构师Agent)
   - 系统设计
   - 技术选型

5. **整合审查** (主Agent)
   - 一致性检查
   - 完整性验证

## 💡 后续建议

1. **用户测试** - 可制作原型进行教师调研，验证需求真实性
2. **MVP规划** - 建议先做2-3个核心功能，快速验证
3. **技术预研** - AI模型选型需实际测试效果
```
