## 系统概览

### 系统定位

**混合动态多 Agent 协作系统**是一个运行在 OpenClaw 之上的通用元级别协作系统，让单一主 Agent 能够动态创建、管理、协调多个临时子 Agent，并行/串行完成复杂任务。

### 核心价值

| 价值 | 说明 | 效果 |
|------|------|------|
| **动态适配** | 不固定子 Agent 数量，按需分配 | 灵活高效 |
| **质量优先** | 三层审查机制，每环节把关 | 输出质量≥85 分 |
| **极致性价比** | 8 大优化模块，Token 节省 90% | 成本降低 95% |
| **持续进化** | 执行经验沉淀，成功流程固化 | 越用越聪明 |
| **完全自主** | 不依赖外部 API，100% 可控 | 安全隐私 |

### 适用场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **创意写作** | 小说、剧本、文章创作 | 10 万字小说 |
| **数据分析** | 报告生成、洞察分析 | 销售分析报告 |
| **代码开发** | 功能开发、重构、测试 | 完整前端项目 |
| **方案设计** | 商业计划、技术方案 | 50 页技术方案 |
| **研究分析** | 市场调研、竞品分析 | 行业研究报告 |

## 工作流程

### 完整工作流程

```
1. 用户输入任务
   ↓
2. 任务分类器判断类型
   ├─ 简单任务 → 直接执行
   ├─ 标准任务 → 动态分解
   └─ 创新任务 → 动态分解
   ↓
3. 任务分解器分解为子任务
   ├─ 确定子任务数量（6-8 个）
   ├─ 确定子任务角色（5-6 个）
   └─ 确定依赖关系
   ↓
4. 子 Agent 管理器创建子 Agent
   ├─ 分配模型（智能选择）
   ├─ 分配职责
   └─ 设置预算
   ↓
5. 执行协调器协调执行
   ├─ 识别可并行任务
   ├─ 管理依赖关系
   └─ 传递上下文
   ↓
6. 质量检查器三层审查
   ├─ 子 Agent 自检
   ├─ 主 Agent 确认
   └─ 审查 Agent 审查（复杂任务）
   ↓
7. Skill 进化分析器
   ├─ 收集执行数据
   ├─ 更新 Skill 计数
   └─ 触发固化（≥3 次成功）
   ↓
8. 资源清理器
   ├─ 删除临时子 Agent
   ├─ 清理临时文件
   └─ 保留固化 Skill
   ↓
9. 返回结果 + 优化统计
```

## 功能模块

### P0 基础优化层（3 个模块）

#### 1. Token 缓存 (core/cache/SKILL.md)

**功能：** 缓存 API 响应，避免重复调用

**核心特性：**
- 精确匹配缓存（SHA256 哈希）
- 语义匹配缓存（嵌入向量）
- 模板缓存（槽位填充）
- LRU 清理策略
- 持久化存储

**节省效果：** 30-60% Token

**使用场景：**
- 重复查询（100% 节省）
- 相似问题（80-90% 节省）
- 模板任务（95% 节省）

---

#### 2. 智能模型选择器 (core/model-selector/SKILL.md)

**功能：** 根据任务复杂度自动选择最优模型

**模型矩阵：**
| 模型 | 能力 | 速度 | 成本 | 适用 |
|------|------|------|------|------|
| MiniMax-M2.5 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ¥ | 简单任务 |
| Qwen3-Coder | ⭐⭐⭐⭐ | ⭐⭐⭐ | ¥¥ | 代码任务 |
| Qwen3.5-Plus | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ¥¥¥ | 标准任务 |
| Qwen3-Max | ⭐⭐⭐⭐⭐ | ⭐⭐ | ¥¥¥¥ | 复杂任务 |

**选择算法：**
```typescript
function selectModel(complexity, budget, urgency) {
  if (urgency === 'high') return 'minimax-m2.5';
  if (complexity < 4) return 'minimax-m2.5';
  if (complexity < 7) return 'qwen3.5-plus';
  return 'qwen3-max';
}
```

**节省效果：** 30-50% Token + 60-80% 成本

---

#### 3. 批量处理器 (core/batch-processor/SKILL.md)

**功能：** 将多个相似任务合并为一次 API 调用

**批量策略：**
- 简单批量（共享提示）
- 结构化批量（JSON 格式）
- 模板批量（槽位填充）

**最优批量计算：**
```typescript
batchSize = min(
  modelContextLimit / avgTokensPerTask,  // 上下文限制
  20,  // 质量上限
  15,  // 超时限制
  tasks.length
)
```

**节省效果：** 50-70% Token + 3-5 倍速度

### P1 高级优化层（2 个模块）

#### 4. 渐进式处理器 (core/progressive-processor/SKILL.md)

**功能：** 分阶段处理（大纲→确认→填充），避免返工

**处理流程：**
```
阶段 1: 生成大纲（500-1000 Token）
   ↓
阶段 2: 用户确认（0 Token）
   ↓
阶段 3: 细节填充（按需分配）
```

**适用场景：**
- 长文写作（>5000 字）
- 方案设计
- 代码开发
- 创意创作

**节省效果：** 50-70% Token + 60-80% 返工减少

---

#### 5. 预算控制器 (core/budget-controller/SKILL.md)

**功能：** 设置 Token 预算，超预算自动告警或降级

**预算模板：**
```typescript
{
  'translation': { base: 500, perWord: 3 },
  'creative_writing': { base: 2000, perWord: 4 },
  'code_generation': { base: 3000, perWord: 5 },
  'analysis': { base: 2500, perWord: 3 },
}
```

**分级告警：**
- 80% → 警告
- 90% → 严重警告
- 100% → 降级或终止

**节省效果：** 40-50% Token + 80% 超支减少

### P2 终极优化层（2 个模块）

#### 6. 智能重试 (core/smart-retry/SKILL.md)

**功能：** API 失败时自动降级模型或切换策略

**重试策略：**
- 指数退避（频率限制）
- 模型降级（超时/服务器错误）
- 任务简化（最后手段）

**降级路径：**
```
Qwen3-Max 失败
   ↓
Qwen3.5-Plus
   ↓
MiniMax-M2.5
   ↓
任务简化
```

**效果：** 92%→99.5% 成功率 + 50-60% 重试成本节省

---

#### 7. 结果复用 (core/result-reuse/SKILL.md)

**功能：** 识别相似任务，复用历史结果

**复用策略：**
| 相似度 | 策略 | 节省 |
|--------|------|------|
| ≥0.95 | 直接复用 | 100% |
| ≥0.85 | 微调复用 | 70-80% |
| ≥0.70 | 部分复用 | 40-60% |

**相似度计算：**
```typescript
combined = semantic * 0.7 + structural * 0.3
```

**节省效果：** 50-60% Token + 35-45% 复用率

### 核心执行层（8 个 v1.0 模块）

#### 8. 任务分类器 (core/task-classifier/SKILL.md)

**功能：** 判断任务类型，选择处理模式

**四种类型：**
| 类型 | 特征 | 处理方式 |
|------|------|----------|
| 简单 (simple) | 单维度、<5 分钟 | 直接执行 |
| 标准 (standard) | 多维度、需协作 | 动态分解 |
| 创新 (innovative) | 复杂、需创意 | 动态分解 |
| 混合 (hybrid) | 标准 + 定制 | 标准 Skill+ 定制角色 |

**识别准确率：** ≥90%

---

#### 9. 任务分解器 (core/task-decomposer/SKILL.md)

**功能：** 将复杂任务分解为可执行子任务

**分解要素：**
- 任务目标（最终要做什么）
- 任务背景（为什么做）
- 任务约束（限制条件）
- 输出要求（格式、规模、质量）

**分解规则：**
| 任务类型 | 维度数 | 子任务数 | 角色数 |
|----------|--------|----------|--------|
| 简单 | 1-2 | 1-2 | 1-2 |
| 中等 | 2-3 | 3 | 3 |
| 复杂 | 4+ | 6-8 | 5-6 |

---

#### 10. 子 Agent 管理器 (core/subagent-manager/SKILL.md)

**功能：** 动态创建子 Agent，分配模型和职责

**创建规则：**
- 专业匹配（子任务类型与角色匹配）
- 负载均衡（避免单一 Agent 过多）
- 依赖优化（有依赖的分给不同 Agent）
- 模型适配（按难度分配模型）

**模型选择策略：**
```
创意写作 → qwen3.5-plus
代码开发 → qwen3-coder-plus
数据分析 → qwen3.5-plus
复杂推理 → qwen3-max
```

---

#### 11. 质量检查器 (core/quality-checker/SKILL.md)

**功能：** 三层质量检查机制

**三层审查：**
```
第一层：子 Agent 自检（所有任务必选）
   ↓
第二层：主 Agent 确认（所有任务必选）
   ↓
第三层：审查 Agent 审查（复杂任务，≥4 子 Agent）
```

**问题分级：**
- critical（致命）- 必须修复
- high（严重）- 应该修复
- medium（中等）- 建议修复
- low（轻微）- 可选修复

- 数据传递（前序输出→后续输入）
