# 女娲 · 造人术

> 「你想蒸馏的下一个员工，何必是同事」

## 核心理念

女娲不是复制人，是**提炼认知操作系统**：

| 层次 | 提取什么 |
|------|---------|
| 怎么说话 | 表达DNA - 语气、节奏、用词偏好 |
| 怎么想 | 心智模型 - 他用什么框架看世界 |
| 怎么判断 | 决策启发式 - 快速判断规则 |
| 什么不做 | 反模式 - 价值观底线 |
| 知道局限 | 诚实边界 - Skill的失效条件 |

---

## 工作流程

### Phase 0: 确认任务

1. 确认人物/主题名字
2. 确认聚焦方向（全面画像 vs 某个维度）
3. 检查 `brain/` 是否已有该人物目录

```
如果 brain/ 下已有 → 询问「更新」还是「新建」
如果用户提供了素材 → 标记为本地语料模式
```

### Phase 1: 信息采集（6路并行）

启动6个子Agent并行采集：

| Agent | 负责 | 搜索方向 |
|-------|------|---------|
| 1-著作 | 书籍、长文、系统性思考 | 反复出现的核心论点、自创术语 |
| 2-对话 | 播客、访谈、AMA | 被追问时的回答方式、即兴类比 |
| 3-表达 | 社交媒体、碎片文字 | 高频用词句式、争议立场 |
| 4-他者 | 书评、批评、传记 | 外部观察、批评、与同行对比 |
| 5-决策 | 重大决策、转折点 | 决策逻辑、事后反思 |
| 6-时间线 | 完整时间线 | 关键里程碑、最新动态 |

**信息源优先级**：本人著作 > 长对话 > 实际决策 > 社交媒体 > 他者评价

**黑名单**：知乎、微信公众号、百度百科（信息失真）

每个Agent结果存入 `brain/[人物名]/research/`

### Phase 2: 框架提炼

从采集结果中提炼：

1. **心智模型**（3-7个）：此人反复使用的思维框架
   - 三重验证：跨域出现、有预测力、排他性
2. **决策启发式**（5-10条）：快速判断规则
3. **表达DNA**：语气、用词、句式特征
4. **反模式**：明确反对的行为/思维
5. **诚实边界**：Skill做不到什么

### Phase 3: 构建Skill

输出3个文件到 `brain/agents/[人物名]/`：

```
brain/agents/[人物名]/
├── persona.md      # 人物画像（心智模型+决策启发式+表达DNA）
├── SKILL.md        # AI角色定义（可直接调用的Skill）
└── conversations/
    └── examples.md # 对话示例
```

**注意**：创建后必须更新 `brain/agents/_registry.json`

### Phase 4: 质量验证

用此人公开表态过的问题测试：
- 方向一致 → 模型有效
- 偏离 → 调整模型

### Phase 5: 交付

展示完成的人物Skill，告知：
- 调用方式
- 适用场景
- 局限边界

---

## 轻量模式（快速蒸馏）

当信息有限时（如冷门人物），缩减为：
- 2-3个心智模型
- 3-5条决策启发式
- 加大诚实边界篇幅
