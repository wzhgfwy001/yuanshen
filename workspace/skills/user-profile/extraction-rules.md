# 用户画像自动提取规则

**版本：** v1.0.0
**更新时间：** 2026-04-17
**状态：** 已实现

---

## 概述

自动提取器从对话消息中分析用户特征，无需人工干预即可构建用户画像。

---

## 提取字段

### 1. communication_style（沟通风格）

| 检测模式 | 关键词/正则 | 权重 | 推断结果 |
|---------|-----------|------|---------|
| 详细展开 | 详细、展开、具体、是什么、为什么 | 0.8 | 用户喜欢详细说明 |
| 简洁回复 | 简洁、简短、扼要、一句话、简明 | 0.8 | 用户喜欢简短回复 |
| 分步说明 | 分步、步骤、第一、第二、逐步 | 0.7 | 用户喜欢分步骤指导 |
| 直接给答案 | 直接给、给我答案、结果是什么 | 0.6 | 用户喜欢直接得到结果 |

**置信度计算：**
- 基础置信度 = 0.5
- 每匹配一次 +0.1
- 最高置信度 = 0.9

---

### 2. preferred_language（语言偏好）

| 语言 | 检测关键词 |
|------|----------|
| 中文 | 中文、汉语、用中文 |
| 英文 | 英文、英语、English |

**默认：** 中文

---

### 3. tech_familiarity（技术熟悉度）

#### 专家指标（出现则加分）
- API、SDK、架构、源码、算法、数据结构
- 异步、并发、多线程、设计模式、重构
- 部署、运维、容灾、缓存、队列

#### 新手指标（出现则加分）
- 什么是、怎么用、入门、基础
- 不太懂、不太会、新手、萌新

#### 级别判定

| 专家指标得分 | 新手指标得分 | 判定级别 |
|------------|------------|---------|
| ≥5 | - | 技术专家 |
| ≥2 | - | 技术熟悉 |
| - | ≥3 | 技术小白 |
| >0 | - | 技术一般 |
| 0 | 0 | 未知 |

---

### 4. work_type（工作类型）

| 类别 | 关键词 |
|------|-------|
| 技术开发 | 代码、开发、编程、程序员、全栈、前端、后端 |
| 产品设计 | 产品、需求、功能、设计、UI、UE、原型 |
| 数据分析 | 分析、数据、报表、统计、可视化 |
| 市场运营 | 运营、推广、营销、增长、用户 |
| 管理 | 管理、团队、项目、协调 |

**判定规则：** 得分最高的类别即为工作类型

---

### 5. decision_style（决策风格）

#### 谨慎决策指标
- 对比、比较、优缺点、利弊、哪个好
- 权衡、考虑、想想、分析一下

#### 快速决策指标
- 直接、赶紧、快点、马上、立刻
- 就这样、行、可以、决定了

#### 风格判定

| 谨慎指标 | 快速指标 | 判定风格 |
|---------|---------|---------|
| > 快速×2 | - | 深思熟虑+多方案对比 |
| - | > 谨慎×2 | 快速决策 |
| >0 或 >0 | >0 或 >0 | 混合型 |

---

### 6. feedback_style（反馈风格）

#### 完美主义指标
- 不对、不对不对、错了、改、修改
- 重写、再来、不满意、不够好

#### 随和指标
- 可以、行、好、不错、挺好、就这样

#### 风格判定

| 完美主义 | 随和 | 判定风格 |
|---------|------|---------|
| > 随和 | - | 完美主义 |
| - | > 完美主义 | 随和 |
| 其他 | 其他 | 未知 |

---

### 7. long_term_goals（长期目标）

| 目标 | 检测关键词 |
|------|-----------|
| 财富自由 | 财富自由、财务自由、赚大钱、有钱 |
| 高考志愿系统 | 高考、志愿、填报 |
| 商业化变现 | 商业化、变现、产品化、上线 |
| 自我提升 | 学习、提升、成长、进步 |
| 健康生活 | 健康、健身、运动、减肥 |

**优先级：** 按提及次数从高到低排序

---

### 8. active_hours（活跃时间）

**分析方法：**
1. 统计每条消息的时间戳
2. 按小时分组计数
3. 按星期分组计数
4. 输出高频时段和活跃日

**输出格式：**
- `typical`: 最活跃时段（如 "09:00-12:00"）
- `days_of_week`: 活跃日数组 [0-6]

---

## 置信度阈值

| 阈值 | 说明 |
|------|------|
| ≥0.9 | 高度确定，可直接采纳 |
| ≥0.7 | 较确定，建议采纳 |
| ≥0.6 | 最低阈值，刚达到采纳标准 |
| <0.6 | 不采纳，仅记录 |

**默认阈值：** 0.6

## 使用方式

```javascript
const { extract, saveResult, loadProfile, mergeProfiles } = require('./auto-extractor');

// 提取用户画像
const result = extract(messages, userId);

// 保存到状态文件
saveResult(result);

// 增量更新已有画像
const oldProfile = loadProfile();
const merged = mergeProfiles(oldProfile, newExtraction);
```
