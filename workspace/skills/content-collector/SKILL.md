# content-collector

**【搜索宝物】Find Treasure** — 

## 功能

1. **读取本地文件** - 提取本地文档内容
2. **分析PDF** - 提取PDF文档关键信息
3. **访问内网服务** - 通过curl访问内网API/系统
4. **内容分析** - 判断内容价值、类别、相关性
5. **结构化输出** - 按统一格式输出，便于后续处理

## 内网兼容性

| 工具 | 状态 | 说明 |
|------|------|------|
| `read` | ✅ | 读取本地文本/文档文件 |
| `write` | ✅ | 写入采集报告 |
| `edit` | ✅ | 编辑/修改文档 |
| `pdf` | ✅ | 分析本地PDF文档 |
| `exec` | ✅ | 执行本地命令（curl访问内网API） |
| `sessions_spawn` | ✅ | 启动子Agent并行采集 |
| `web_search` | ⏸️ | 可选 - 有网络时可用 |
| `web_fetch` | ⏸️ | 可选 - 有网络时可用 |
| `browser` | ⏸️ | 可选 - 仅内网URL或本地文件 |

## 内网替代方案

### 方案1：本地文件采集（推荐）
```
用户提供本地文档/文件夹路径 → 使用 read 读取 → 整理分析
```

### 方案2：PDF文档采集
```
用户提供PDF文件路径 → 使用 pdf 分析 → 提取关键信息
```

### 方案3：内网API采集
```
用户提供内网API地址 → 使用 exec + curl 访问 → 解析返回数据
```

### 方案4：混合模式
```
本地文件 + 内网服务 → 综合整理
```

## 工作流程

### 内网环境工作流（当前版本）
```
用户需求 → 用户提供资料（本地文件/内网URL） → 读取/抓取 → 分析整理 → 输出报告
```

### 搜索本地文件内容
```
用户提供关键词 → 使用 exec + Select-String/grep 搜索 → 整理结果
```

## 本地文件采集策略

| 文件类型 | 工具 | 说明 |
|---------|------|------|
| .txt/.md/.json | `read` | 直接读取文本内容 |
| .pdf | `pdf` | 分析提取文本 |
| .docx/.xlsx | `exec` + 工具 | 需要安装相应工具 |
| 文件夹 | `exec` + dir/ls | 遍历读取所有文件 |

## 内网API采集

### 访问内网REST API
```powershell
# GET请求
curl -Method GET "http://internal-api.example.com/data"

# POST请求
curl -Method POST -Body '{"key": "value"}' "http://internal-api.example.com/api"
```

### 内网系统类型
- 学校教务系统
- 公司内部数据库
- 政务内网系统
- 自建内网服务

## 内容分析框架

分析采集的内容时，评估：

| 维度 | 评估项 |
|------|--------|
| 相关性 | 与目标主题的关联程度 |
| 完整性 | 内容是否完整 |
| 时效性 | 内容更新时间 |
| 价值性 | 是否有参考价值 |
| 可借鉴性 | 改写/利用空间 |

## 内网使用建议

1. **提供本地资料路径** - 明确的文件夹或文件路径
2. **提供内网URL** - 内网API或系统的访问地址
3. **明确采集目标** - 说明想获取什么主题的内容
4. **准备认证信息** - 如内网API需要认证，提供相应信息
