# 《吾名午夜》小说工作流规范

**创建时间：** 2026-04-18
**版本：** v1.0
**依据：** 用户提供的7点建议 + 原有6阶段流程

---

## 📝 工作流6阶段

```
立项 → 故事大纲 → 剧情 → 内容 → 审核 → 去AI化
```

| 阶段 | 说明 |
|------|------|
| 1. 立项 | 确定作品方向、核心主题 |
| 2. 故事大纲 | 完整的故事架构 |
| 3. 剧情 | 具体情节安排 |
| 4. 内容 | 正文创作 |
| 5. 审核 | 质量检查 |
| 6. 去AI化 | 消除AI味 |

---

## ⭐ 7条核心要求

### 1️⃣ 金点子系统
- 用户可以添加金点子（创意、转折、设定）
- 用户决定金点子出现在哪些时间线和场景
- 系统负责落实和衔接

### 2️⃣ 时间线闭环
- 故事逻辑要自洽
- 时间线要完整
- 开头和结尾要形成闭环
- 伏笔必须回收

### 3️⃣ 人物场景一致性
- 人物设定好后，身形、脸部、穿着打扮等细节
- 全书保持一致性
- 合理性（符合角色背景和性格）
- 需要建立人物档案卡

### 4️⃣ 去AI味（强制）
- 内容创作必须去除AI味
- 参考：brain/mao-zedong/SKILL.md 中的去AI味规范
- 参考：skills/dynamic-multi-agent-system/novel-wuming/roles/tone-editor/LEARNING.md

### 5️⃣ 风格学习能力
- 能够学习并模仿不同写手/小说家的写作风格
- 存储风格特征到文件
- 需要时调用对应风格

### 6️⃣ 持续学习用户风格
- 根据用户自己的写作习惯和风格
- 越来越像用户自己写的
- 需要建立用户风格档案
- 每次创作后收集用户反馈并迭代

### 7️⃣ 拆解改编能力
- 能够分析并拆解别的作品
- 分析内容：故事大纲、剧情转变、人物塑造
- 具备改编能力
- 可以将其他作品改编为自己的故事
