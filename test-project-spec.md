# 项目规格文档：个人时间追踪应用

## 1. 核心价值（Core Value）

帮助知识工作者清晰地看见自己的时间花在哪里，从而做出更明智的时间分配决策。

## 2. 问题陈述（Problem Statement）

大多数人对"时间都去哪儿了"只有模糊的感知。每天忙忙碌碌，但到了周末回顾时，发现真正重要的任务没做几件。缺乏对自己时间分配的量化认知，是时间管理失效的根本原因。

## 3. 需求（Requirements）

### 必须有（Must Have）
- 手动记录任务开始/结束时间
- 按项目/标签分类时间条目
- 每日/每周/每月时间报告（饼图+柱状图）
- 数据本地存储（JSON文件）

### 应该有（Should Have）
- 番茄钟模式（25分钟专注+5分钟休息）
- 目标设定（每周X小时用于某项目）
- 周报自动生成（文字版）

### 不应该有（Should Not Have）
- 自动时间追踪（侵犯隐私，学习成本高）
- 云同步（增加复杂度，与核心价值无关）
- 团队协作功能（偏离个人工具定位）

## 4. 约束（Constraints）

- 桌面端应用（Windows/macOS）
- 单用户本地使用，无需注册
- 数据格式：JSON（便于备份和迁移）
- 技术栈：Electron + React

## 5. 关键决策（Key Decisions）

| 决策 | 选择 | 备选 | 放弃原因 |
|------|------|------|---------|
| 界面风格 | 极简主义 | 丰富仪表盘 | 减少认知负担，聚焦核心 |
| 数据存储 | 本地JSON | SQLite | 便于用户自行备份和查看 |
| 报告周期 | 日/周/月 | 年/季度 | 对个人用户足够 |

## 6. 参考点（Reference Points）

- **类似产品**：Toggl（太复杂）、RescueTime（自动追踪不适合）
- **设计参考**：Linear的简洁感
- **使用场景**：自由职业者、程序员、学生的每日复盘
