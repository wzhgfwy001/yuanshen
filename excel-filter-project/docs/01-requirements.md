# Excel 筛选查找器 - 需求分析文档

## 一、详细功能列表

### 1. 核心功能

#### 1.1 数据导入
- [ ] 支持导入 Excel 文件 (.xlsx, .xls)
- [ ] 支持导入 2 个表格：
  - 学校/专业信息表（包含学校名称、专业、选科要求、历年分数线等）
  - 学生信息表（包含学生姓名、选科、分数、排名等）
- [ ] 自动识别表格列名和数据类型
- [ ] 数据预览功能（导入前预览前 10 行）
- [ ] 数据校验（检查必填字段、数据格式）

#### 1.2 选科条件筛选
- [ ] 支持物理/历史类选择
- [ ] 支持再选科目筛选（化学、生物、政治、地理）
- [ ] 支持"必选"、"可选"、"不限"三种选科要求匹配
- [ ] 支持多科目组合筛选（如：物理 + 化学）

#### 1.3 分数/排名筛选
- [ ] 按高考总分筛选
- [ ] 按省内排名筛选
- [ ] 支持分数区间筛选（最低分 - 最高分）
- [ ] 支持排名区间筛选
- [ ] 支持"冲稳保"三档自动划分：
  - 冲：分数略低于往年录取线（-10 分以内）
  - 稳：分数接近往年录取线（±5 分）
  - 保：分数高于往年录取线（+10 分以上）

#### 1.4 可报学校/专业列表生成
- [ ] 生成符合条件的学校列表
- [ ] 生成符合条件的专业列表
- [ ] 显示每个学校/专业的详细信息：
  - 学校名称、所在地、办学层次（985/211/双一流）
  - 专业名称、选科要求、历年分数线、招生人数
- [ ] 支持按多个维度排序（分数、排名、学校层次、地域）
- [ ] 支持导出结果（Excel、PDF）

#### 1.5 条件组合筛选
- [ ] 支持多条件 AND/OR 逻辑组合
- [ ] 支持地域筛选（省份、城市）
- [ ] 支持学校层次筛选（985、211、双一流、普通本科）
- [ ] 支持专业类别筛选（理工、文史、医学、艺术等）
- [ ] 支持保存筛选条件模板

### 2. 辅助功能

#### 2.1 数据更新
- [ ] 支持手动更新数据（导入新 Excel）
- [ ] 支持数据版本管理
- [ ] 支持数据备份和恢复

#### 2.2 用户界面
- [ ] 简洁直观的操作界面
- [ ] 筛选条件可视化配置
- [ ] 结果列表分页显示
- [ ] 搜索功能（按学校名、专业名搜索）

#### 2.3 帮助与支持
- [ ] 使用教程
- [ ] 常见问题解答
- [ ] 数据格式模板下载

## 二、用户场景分析

### 场景 1：高考志愿填报（主要场景）

**用户画像：**
- 高三学生或家长
- 刚参加完高考，有明确分数和排名
- 对志愿填报规则不太熟悉
- 需要快速筛选出可报考的学校和专业

**使用流程：**
1. 下载并导入官方发布的学校/专业数据表
2. 输入自己的高考分数和排名
3. 选择自己的选科组合
4. 设置筛选条件（地域、学校层次等）
5. 查看筛选结果，按"冲稳保"分类
6. 导出结果，进行进一步比较和选择

**痛点：**
- 手动筛选大量数据耗时耗力
- 容易遗漏符合条件的学校
- 不清楚"冲稳保"的合理划分

### 场景 2：模拟考试志愿预演

**用户画像：**
- 高三学生（一模、二模后）
- 想提前了解自己的目标学校
- 需要制定学习计划

**使用流程：**
1. 导入模拟考试成绩
2. 设置期望的学校层次和地域
3. 查看当前成绩可报的学校
4. 对比目标学校的分数差距
5. 制定提分计划

### 场景 3：升学规划咨询

**用户画像：**
- 高中老师、升学顾问
- 需要为多个学生提供志愿建议
- 需要批量处理学生数据

**使用流程：**
1. 批量导入多个学生信息
2. 为每个学生生成可报学校列表
3. 对比分析不同学生的选择
4. 导出报告给学生和家长

## 三、数据结构设计

### 3.1 学校/专业信息表 (schools.xlsx)

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| school_id | 文本 | 是 | 学校唯一标识 | SCH001 |
| school_name | 文本 | 是 | 学校名称 | 清华大学 |
| province | 文本 | 是 | 所在省份 | 北京 |
| city | 文本 | 是 | 所在城市 | 北京 |
| level | 文本 | 是 | 办学层次 | 985/211/双一流 |
| type | 文本 | 是 | 学校类型 | 理工/综合/师范 |
| major_id | 文本 | 是 | 专业唯一标识 | MAJ001 |
| major_name | 文本 | 是 | 专业名称 | 计算机科学与技术 |
| major_category | 文本 | 是 | 专业类别 | 理工 |
| subject_requirement | 文本 | 是 | 选科要求 | 物理 + 化学 |
| subject_required | 文本 | 否 | 必选科目 | 物理，化学 |
| subject_optional | 文本 | 否 | 可选科目 | 生物，地理 |
| year | 数字 | 是 | 招生年份 | 2025 |
| min_score | 数字 | 是 | 最低录取分 | 680 |
| min_rank | 数字 | 是 | 最低录取排名 | 100 |
| avg_score | 数字 | 否 | 平均录取分 | 685 |
| enrollment_count | 数字 | 否 | 招生人数 | 50 |
| tuition | 数字 | 否 | 学费（元/年） | 5000 |

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| student_id | 文本 | 是 | 学生唯一标识 | STU001 |

| category | 文本 | 是 | 考生类别 | 物理类/历史类 |

| preference_level | 文本 | 否 | 期望层次 | 985,211 |
| preference_major | 文本 | 否 | 期望专业类别 | 理工 |

### 3.3 筛选结果表 (results.xlsx)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| student_id | 文本 | 学生 ID |
| student_name | 文本 | 学生姓名 |
| school_id | 文本 | 学校 ID |
| school_name | 文本 | 学校名称 |
| major_id | 文本 | 专业 ID |
| major_name | 文本 | 专业名称 |

## 四、非功能性需求

### 4.1 性能要求
- 单次筛选响应时间 < 3 秒（1 万条学校数据）
- 支持同时处理 100+ 学生数据
- 内存占用 < 500MB

### 4.2 兼容性要求
- Windows 10/11
- 微信小程序（基础库 2.19.0+）
- 支持 Excel 2007+ 格式

### 4.3 数据安全
- 本地存储，不上传学生个人信息
- 支持数据加密（可选）
- 操作日志记录

### 4.4 可用性
- 界面简洁，无需培训即可使用
- 提供数据模板和示例
- 错误提示清晰友好
