## llm-wiki-sync v2.0.0 功能规格（重建参考）

### 核心操作
| 操作 | 描述 |
|------|------|
| compile | JSON/Markdown编译检查 |
| lint | LLM Wiki 7项自检 |
| sync | OpenClaw↔Obsidian 数据对齐 |
| index | 索引统计 |
| ingest | 导入内容到Wiki |
| query | 搜索Wiki回答 |
| reindex | 重建index.md |
| health | 健康评分(0-100) |
| dedup | 查重扫描 |
| stats | 详细统计面板 |
| backup | 备份工作区 |
| export | 导出报告(html/md/json) |

### 新增参数
- `--format, -f html|md|json` 导出格式
- `--interactive, -i` 交互模式
- `--watch, -w` 监视模式
- `--interval, -t <秒>` 监视间隔
- `--out, -o <路径>` 导出输出路径
- `--backup-dir <路径>` 备份目录
- `--verbose, -v` 详细输出

### 配置
```javascript
CONFIG = {
  ROOT: 'C:/Users/DELL/.openclaw/workspace',
  OBSIDIAN_ROOT: 'D:/obsidian知识库/我的知识库',
  WIKI_DIR: 'D:/obsidian知识库/我的知识库/wiki',
  BACKUP_DIR: 'D:/obsidian知识库/备份',
  EXPORT_DIR: 'C:/Users/DELL/.openclaw/workspace/llm-wiki-reports'
};
```

### 健康评分算法 (health)
- 文件完整性: -20 if empty
- 目录结构: -10 per missing (concepts/entities/sources)
- index.md存在: -15 if missing
- 短笔记比例>20%: -10
- 孤立页面>30%: -10
- decisions不同步: -5
- lessons不同步: -5

### 输出格式
- 颜色: g(绿)/r(红)/y(黄)/c(青)/b(蓝)/m(紫)
- hr() 标题分隔
- log(msg, type) 带图标日志

### 关键函数
- scanWiki(dir) - 递归扫描.md文件
- parseLinks(content) - 解析[[link]]格式
- readFile(p) / exists(p) / ls(p) - 文件工具
- readIndex() - 读取index.md
